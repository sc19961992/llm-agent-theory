## 二、核心问题与发现

### 核心问题

代码生成任务中，Agent通常通过迭代的生成-验证循环来改进代码。然而，**强化学习（RL）训练在这一场景中的应用面临两个关键挑战**：(1) 稀疏的结果奖励（outcome reward）无法有效引导多步推理过程；(2) Agent的自我验证（self-verification）能力不可靠，容易被奖励破解（reward hacking）。如何让代码Agent通过RL训练真正学会"生成代码-验证代码-修正代码"的完整循环？

### 主要发现

1. **显式优化自我验证能力是关键**。将验证作为独立的目标进行优化（而非仅仅优化代码生成），可以显著提升Agent的长期推理能力。

2. **仅训练3轮推理，即可在推理时扩展到20+轮**。经过ReVeal框架训练的模型，在测试时展现出持续的自我改进能力（从第1轮的34.8% Pass@1持续提升到第25轮的38.7%）。

3. **零退化（zero degradation）**。ReVeal训练后的模型在修正过程中，初始正确的答案不会因为后续的过度修正而变错（Δ↓ = 0.0%），这与未经专门训练的模型形成鲜明对比。

4. **超越DeepSeek-R1-Zero-Qwen-32B**。以DAPO-Qwen-32B为基座模型，仅用11k TACO训练数据，就在竞技编程基准上超越了强推理基线。

## 三、核心方法/框架

### 3.1 ReVeal框架概览

ReVeal是一个**多轮强化学习框架**，将代码Agent的长程推理结构化为**交替的生成回合（Generation Turn）和验证回合（Verification Turn）**：

```
循环结构：
  生成回合 → [代码候选] → 验证回合（构造测试+工具执行）→ [反馈] → 下一轮生成回合 → ...
```

- **生成回合**：模型生成候选代码
- **验证回合**：模型自我构造测试用例，调用外部工具（Python解释器）执行，返回执行结果和反馈
- **不依赖预定义测试集**：所有测试由模型自主生成

### 3.2 TAPO：Turn-Aware Policy Optimization（回合感知策略优化）

TAPO是本文的核心技术贡献，它替代了标准PPO中的GAE（广义优势估计）：

**传统PPO的局限**：
- 使用GAE进行信用分配（credit assignment），对所有token使用统一的折扣因子
- 无法区分"生成代码"和"验证代码"这两个性质不同的回合
- 容易导致reward gaming——模型可能生成琐碎代码来破解验证奖励

**TAPO的创新**：
- **Token级回报**：蒙特卡洛回报（γ=1, λ=1），保留了每步的完整信息
- **回合级回报（Turn-level Return）**：在回合边界处进行信用分配的重新校准
  - 生成回合的奖励仅基于代码质量（而非验证成功）
  - 验证回合的奖励仅基于验证质量（而非生成成功）
- 通过回合分割避免reward gaming，确保两种能力被独立优化

### 3.3 联合可验证奖励（Joint Verifiable Rewards）

| 奖励组件 | 描述 | 作用 |
|---------|------|------|
| **结果奖励（Outcome Reward）** | 最终解决方案质量（通过率+格式） | 衡量整体任务完成度 |
| **生成奖励（Generation Reward）** | 每轮代码准确性和改进程度 | 鼓励持续的代码改进 |
| **验证奖励（Verification Reward）** | 生成的测试在标准解上的通过比例 | 鼓励构造有质量的测试 |

### 3.4 训练配置

- **基座模型**：DAPO-Qwen-32B（在数学数据上强化过的Qwen2.5-32B）
- **训练数据**：11k条TACO（代码竞赛数据）
- **训练回合数**：仅3轮生成-验证回合
- **推理回合数**：扩展到25轮
- **RL框架**：基于PPO的TAPO

### 3.5 实验结果

**LiveCodeBench V6 (2025.02-2025.05) — Pass@1**

| 模型 | Pass@1 |
|------|--------|
| Qwen2.5-32B-Instruct | 24.8% |
| Qwen2.5-Coder-32B-Instruct | 29.5% |
| DAPO-Qwen-32B（基座） | 31.1% |
| 单轮RL（仅结果奖励） | 32.8% |
| ReVeal ×25轮推理 | **38.7%** |

**CodeContests — Pass@1**

| 模型 | Pass@1 |
|------|--------|
| Qwen2.5-32B-Instruct | 13.3% |
| Qwen2.5-Coder-32B-Instruct | 14.6% |
| DAPO-Qwen-32B（基座） | 18.5% |
| 单轮RL（仅结果奖励） | 21.0% |
| ReVeal ×25轮推理 | **33.6%** |

**关键对比**：
- 单轮RL（仅优化代码生成，无验证优化）在LCB上仅达32.8%，而ReVeal达38.7%
- CodeContests上差距更大：单轮RL为21.0%，ReVeal为33.6%（+12.6点）
- 与DeepSeek-R1-Zero-Qwen-32B相比，ReVeal以更少的训练数据和更简单的架构取得了更优结果

## 四、关键结论

1. **验证能力是可以被显式训练的**。将self-verification作为独立的RL优化目标，而非仅仅依赖隐式的涌现，可以大幅提升代码Agent的迭代改进能力。

2. **回合级信用分配是训练多轮Agent的关键**。TAPO通过回合感知的优势估计，解决了传统PPO在多轮交互场景下的reward gaming问题。

3. **训练-推理回合数差距（3轮→25轮）表明**：RL训练学到的是"改进的能力"（how to improve），而非固定的改进步骤数（how many times）。这正是生成-验证循环的核心价值——它习得了一种可以持续迭代的元能力。

4. **零退化属性至关重要**：在代码修正场景中，确保正确的输出不会被后续迭代破坏，是实际部署的前提条件。

## 五、与"AI原理草稿"的关系

### 草稿核心论点2：Agent依赖"测试收敛"——由可验证目标驱动的、通过反复生成-验证循环向目标逼近的过程，但测试收敛不被任何数学保证所覆盖

**论文的观点：**

ReVeal是**草稿"测试收敛"概念最直接的经验验证案例**之一，两者之间存在深刻的对应关系：

**（1）高度一致：**

- **生成-验证循环**：ReVeal的核心结构正是草稿所述的"生成-验证循环"——生成候选代码、自我构造测试、执行验证、根据反馈修正，构成闭环
- **可验证目标**：代码任务天然具有可验证性（编译通过、测试通过），恰好对应草稿强调的"可验证目标驱动"
- **迭代性逼近**：Pass@1从34.8%（第1轮）提升到38.7%（第25轮），体现了"反复生成-验证循环向目标逼近"的过程
- **训练-推理的深层一致性**：草稿认为Agent的核心能力不是在训练时收敛，而是在测试时通过循环收敛——ReVeal恰好展示了这一点：仅训练3轮，却在推理时扩展到25轮

**（2）对"测试收敛不可保证性"的验证与挑战：**

- **验证**：ReVeal确实无法保证收敛——在第25轮后性能进入平台期，虽然不退化，但可能也无法进一步提升。这印证了草稿"测试收敛不被任何数学保证所覆盖"的判断。
- **挑战**：ReVeal通过RL训练大幅缩小了"需要搜索的空间"，使生成-验证循环在代码任务上变得高效且可靠。这表明，**虽然总体上无法保证收敛，但通过在分布内数据上进行RL训练，可以在特定任务领域上显著提高收敛的概率和效率**。

**（3）更深层的分析：TAPO与测试收敛的关系**

TAPO的核心机制——回合级信用分配——本质上是在优化生成-验证循环的**内部结构**：

- **生成奖励**只奖励代码质量 → 迫使生成器在验证反馈的基础上做有意义的改进
- **验证奖励**只奖励测试质量 → 迫使验证器生成真正有判别力的测试

这种分离式优化解决了草稿提到的"验证者-生成者循环可能退化为自指循环"的问题——如果验证和生成的奖励纠缠不清，Agent可能学会生成"容易验证的代码"而非"正确的代码"。TAPO的回合分离巧妙地避免了这种退化，使得生成-验证循环能持续产生有效的改进信号。

**（4）与"测试收敛"的核心差异：**

| 维度 | 草稿"测试收敛" | ReVeal的实际表现 |
|------|---------------|-----------------|
| **分析层次** | 理论极限分析——追问在无限计算下能否保证收敛 | 工程实践——展示在有限计算下如何有效逼近 |
| **收敛保证** | 强调"无保证"——结构限制是固有的 | 实际收敛——在特定任务上表现稳定且可预测 |
| **失败模式** | 关注自指退化、无限循环、发散 | 观察到零退化和自然平台期，但无发散 |
| **解释** | 递归结构导致不完备性 | RL训练使生成-验证循环被"驯化"为高效搜索 |
| **根本分歧** | 测试收敛是**结构限制下的必然结果** | 测试时缩放是**工程优化下的有效策略** |

## 六、一句话评价

ReVeal是草稿"测试收敛"概念在代码Agent领域最直接的经验验证——通过RL训练将生成-验证循环从"基本可用"优化为"高效且持续改进"，但最终仍面临收益递减的平台期，证实了测试收敛的经验有效性而非理论可保证性。
