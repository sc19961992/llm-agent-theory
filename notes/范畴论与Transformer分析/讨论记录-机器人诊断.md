# 机器人 AI 化的 SSD 诊断

> 从 S/V 分离框架出发，机器人 AI 的根本瓶颈和架构方案。

---

## 一、假瓶颈

| 常被说的瓶颈 | 为什么不是根本瓶颈 |
|---|---|
| **数据不够** | 视觉 S 可用视频+仿真无限训。贵的只有任务特定的 V 数据。S/V 分离后 V 真实数据量可降一两个数量级 |
| **硬件太贵** | 力控传感、灵巧手在快速降价。十年前 GPU 也贵——成本会被规模摊平 |
| **sim-to-real gap** | 感知端靠大规模视觉预训练弥合。动力学端：离散动作码 + 在线 V 更新，仿真只做 S 初始训练，V 在真实机器人上在线调 |
| **安全** | V 在线更新 → 错了 → 接触反馈 → 立刻调策略 → 下次不犯。V 在线是安全机制，不是安全隐患 |

---

## 二、真瓶颈：三样东西锁死了

### 瓶颈一：S 和 V 的部署后冻结

当前所有机器人模型（RT-2、Octo、π0）部署后**全部权重冻结**——训练时 S 和 V 一起训，部署后两者一起冻。机器人到了新环境、新任务后模型是死的——它知道什么是杯子（S 训过），但不知道**这个具体的杯子多重、表面滑不滑**（V 需要在线试一下）。

> **S 应该离线训好冻结。V 应该永远在线更新。** 机器人的核心学习不是"杯子是什么"（S 管），是"这个杯子的重量 × 这个手指的摩擦力 = 多少抓取力"（V 管）。后者不可能在训练集里穷举——每个杯子、每只手、每个角度都不一样。

### 瓶颈二：接触没有独立表示

注意力输出的是信息——"我应该读哪个物体"。接触输出的是力——"我应该对这个物体施加多少力"。

```
信息注意力：token A attend token B → 读取 B 的信息
接触注意力：手 attend 杯子 → 输出力向量 f
```

数学上同构（都是加权和），物理上不同（信息 vs 力）。当前架构压进了同一个 Attention 机制——手 attend 杯子时，模型不知道这是在"读信息"还是在"施加力"。

> **接触是 A 的物理模式——不是信息路由，是力路由。** 这个区分不在架构里 → 抓取不稳定、接触不可控。这是灵巧手操作几乎都在实验室的原因。

### 瓶颈三：动作的连续 vs 离散之争没有从任务定义出发

RT-2 token 化动作（连续 → bins），π0 输出连续值，ACT 用动作码本。每种方案都在说"更好"，但没有从任务定义出发问：**什么粒度的离散化对 V 的在线更新最合适？**

V 需要在线试几次就学会新接触力 → 动作空间不能太大。M = 500-1000 的动作码本，V 试 10 次足够收到反馈信号覆盖动作空间。连续动作空间 → V 试 100 次才能找到方向。

> **离散是 V 在线学习的结构前提——不是偏好，是条件。**

---

## 三、根本瓶颈就一个

> **部署后模型是死的。** 机器人到了新环境不能继续学——因为是端到端，端到端把需要冻的（S）和需要活的（V）锁在了同一组权重里。

解决方向不是更多数据、更好硬件、更好仿真——是**把 S 和 V 分开训练、分开部署**：S 离线训好冻结，V 部署后永远在线。

---

## 四、SSD 机器人架构

```
离线阶段（训 S = 骨架，冻结）：
  大规模视频 + 仿真数据
        ↓
  ┌─────────────────────────┐
  │ S1. 物体槽提取器         │  ← Slot Attention，训一次冻结
  │     像素 → K 个物体 token  │
  ├─────────────────────────┤
  │ S2. 动作码本             │  ← M 类运动基元，训一次冻结
  │     物体轨迹 → 离散动作码  │
  ├─────────────────────────┤
  │ S3. 自躯干编码器         │  ← 不需训——关节角度+运动学直接输入
  │     已知物理量            │
  └─────────────────────────┘

在线阶段（持续训 V = 策略）：
  当前任务 + 当前状态
        ↓
  ┌─────────────────────────┐
  │ V1. 交互路由器           │  ← 小型 Transformer，在线持续更新
  │     物体槽 ↔ 自躯干       │     "手该碰杯子还是躲开人"
  │     → 接触目标 + 力       │
  ├─────────────────────────┤
  │ V2. 动作选择器           │  ← 从 M 个动作码中选一个
  │     A(自躯干, 目标物体)   │     用 RL 信号在线更新
  │     → a_t ∈ {1..M}       │
  └─────────────────────────┘
        ↓
  物理解码器（S3 + a_t → 关节指令）← 不需学习
        ↓
  执行 → 观测新状态 → 更新物体槽 → 更新 V
```

### 关键差异

| | 端到端（RT-2/Octo/π0） | SSD 做法 |
|---|---|---|
| **S 和 V 是否分离** | 否——全部在一个模型里 | **是——S 冻结，V 在线更新** |
| **部署后是否更新** | 否——全部权重冻结 | **是——只有 V 的参数更新** |
| **物理是否被学习** | 是——从数据中学物理 | **否——运动学直接编码，接触力学在物理解码器里** |
| **动作表示** | 连续关节值或 bins | **离散动作码 → 解码为连续指令** |
| **训练数据成本** | 需大量真实机器人数据 | **S 用视频+仿真（便宜），V 少量真实数据在线训** |

## 五、机器人在 SSD 中的特殊位置

| | 语言 | 视频 | 机器人 |
|---|---|---|---|
| **S 骨架来源** | 语法（弱） | 空间邻近+时间连续性（中） | **自躯干+物理+运动学（最强）** |
| **V 推理负担** | 重——所有语义推理靠 A | 中等——物理分担了一部分 | **最轻——大量推理被 S 吸收** |
| **外层完备性** | softmax 天然完备 | 需扩散补 | **不需要——状态预测闭环不需要归一化采样** |
| **部署后 V 更新** | 当前不做 | 当前不做 | **必须做——接触参数无法穷举** |
| **有效 L（推理深度）** | 高 | 中 | **最低——S 最强所以 V 最浅** |

> **机器人是 SSD 框架中 S 骨架最完整的模态。这反而让机器人成了验证 SSD 的最短路径：在 S 骨架最强的地方，V 的推理负担最轻。机器人的有效推理深度应显著低于同规模的语言模型。**

---

## 六、最短验证路径

不是造完整机器人。是验证**"S 冻结 + V 在线更新"是否比"全部冻结"在新任务上适应更快**：

1. 拿已有端到端模型（如 Octo），仿真中测新任务适应曲线
2. 分两组：A 组全部权重冻结只靠 prompt。B 组冻结主体（S），只开放最后几层 Attention 权重 + 动作输出层在线用 RL 更新（V）
3. 对比任务成功率随新数据量的上升曲线

**SSD 预测**：B 组的适应速度显著快于 A 组，需要的任务数据量更少——因 S 不需重训，只有路由策略需要更新。

> 如果此预测成立，端到端路线的经济学前提（"所有任务都需要全部参数"）就瓦解了。
