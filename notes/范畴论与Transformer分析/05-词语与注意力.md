# 词语与注意力——路由，不是近似

## 核心命题

> **注意力不是在近似词语间的范畴。它是在 ℝᵈ 语义空间中为每个 token 选择移动方向的机制。**

---

## 两个结构的并置

| | 词语蕴涵 | 注意力 |
|---|---|---|
| 关系 | 狗 → 动物（蕴涵） | a_{ij} = softmax(Q_i · K_jᵀ) |
| 性质 | 传递 ✓ | 传递 ✗ |
| 性质 | 不对称 ✓ | 不对称（行归一化破坏对称性） |
| 性质 | 稀疏 | 稠密 |
| 性质 | 静态（词典事实） | 动态（每个上下文逐句变化） |
| 性质 | 定义性 | 工具性 |
| 结构 | 预序范畴（hom 是 {*} 或 ∅） | 非范畴（hom 是 [0,1] 中的数，无复合） |
| 米田 | 精确成立 | 不适用 |
| 作用域 | 语义空间 ℝᵈ 内部的关系 | 在 ℝⁿ 上做路由决策，结果落在 ℝᵈ |

---

## 注意力不近似词语蕴涵

注意力权重 a_{ij} 不近似 hom(词_j, 词_i)：

```
"狗在追猫"
  注意力的 a_{ij} 可能让"追"高度关注"狗"和"猫"（论元结构）
  但"狗 → 追"不是蕴涵，"追 → 猫"也不是
  注意力捕捉句法-语义角色绑定，不是上下位关系
```

蕴涵关系存储在参数中（通过 `P(动物 | ...狗...)` 在输出端检测），不在注意力矩阵里。

---

## 注意力在做什么

> "在当前上下文中，构建 token i 在语义空间中的新位置时，应该从 token j 取多少信息——以及往哪个方向走。"

注意力是**路由决策**：它决定 token i 的轨线在 ℝᵈ 中朝什么方向走这一步。

```
Δx_i = MLP( Σ_j a_{ij} · V(x_j) )

a_{ij} 选择"朝哪些 token 的方向走"
V(x_j) 给出"那些方向是什么"
MLP 给出"这些方向叠加后的非线性修正"
```

每一步走完后，所有 token 在 ℝᵈ 中的位置都变了 → 下一层的注意力面对的是一个新的语义配置 → 路由决策也变了。

**注意力不是在固定空间中做推断。它在每一步之后改变空间本身——这正是它和条件概率推断的本质区别。**

---

## 为什么非传递性是功能性的

注意力不满足传递性：`a_{ik} ≠ something(a_{ij}, a_{jk})`。从范畴论看这是缺陷（不构成复合）。从聚类动力学看，这是功能性的：

如果注意力满足传递性，token i 关注 token k 的方式被 i→j 和 j→k 约束。这等价于强制注意力在低维线性子空间上运作——**传递性 = 约束 = 聚类自由度被削减。**

非传递性 = 每对 token 的注意力独立于其他 token 对 = 聚类结构可以任意复杂。

> **注意力的非传递性不是需要修复的设计缺陷。它是 ℝⁿ 端路由能够产生复杂语义聚类的数学前提——与 ℝᵈ 端 W 矩阵的复合（传递）成互补。**
> 
> **一方传递（语义空间），一方不传递（路由空间）。双方正交共存 = 语义结构可以在空间上形成，且路由可以自由探索它。**

---

## 在范畴论中的角色

注意力矩阵 A 可以放进范畴论，但不作为词语范畴的态射：

| 范畴论角色 | 是什么 |
|-----------|-------|
| ℝᵈ 端：Vect 的自同态 | W: ℝᵈ → ℝᵈ（语义空间上的固定变换） |
| ℝⁿ 端：路由算子 | A: 当前语义配置 → 每个 token 的移动方向 |
| 算子，非态射 | A 更新的是"下一步往哪走"，不是断言"token 间是什么关系" |

注意力是作用于语义空间轨线的算子，词语范畴的态射是语义空间内部的固定关系。两者作用的域和方式完全不同。

---

## 映射链条：从词语到注意力再到轨线

```
词语层    蕴涵范畴（预序，米田成立）
  │  φ: 不是函子——分词切割语义单位
  ↓
Token 层  离散标识符序列（无范畴结构）
  │  Embedding: 映射到 ℝᵈ
  ↓
表示层    x_i^(l) ∈ ℝᵈ（有 Vect 态射结构）
  │  注意力: Q·Kᵀ 构造 A^(l)，决定路由
  │  MLP + 残差: 执行移动
  ↓
表示层    x_i^(l+1) ∈ ℝᵈ（语义空间中的新位置）
  │  下一层面对的是被改变后的语义空间
  │  路由决策因此逐层变化
  ↓
  ... 重复 L 次 ...

输出层    最终语义空间配置 → 解码为 token 分布
```

**注意力的角色：不是断言关系，不是近似蕴涵，不是条件概率推断。它是语义空间动力系统的每一步的"选路器"——在 ℝⁿ 上做决定，在 ℝᵈ 中着陆，然后空间本身被更新。**
