# 具体模型的 SSD 诊断

> S = 骨架（持久载体、语义约束）| V = 变异（局部自由、不可归约操作）

---

## 一、图像模型

### 1. ViT（有监督）

```
图像 → 切 16×16 patch → 线性投影 → + 位置编码 → 残差流 → L 层 Transformer → 分类头
```

**S 骨架**：位置编码 + patch 间的空间邻近关系。纯几何，非语义。两个相邻 patch 可能属于同一物体也可能不属于——空间邻近不是语义约束。

**V 变异**：全对全 Attention。有效——但头几层在模拟 CNN 的低级特征提取（边缘、纹理），而不是语义路由。

**诊断**：S/V 比例在底层严重偏低。**底层 A 的行为是"特征聚合器"，不是"路由器"**——这跟文本 Transformer 的根本差异：文本每一层都在做语义路由，图像前 1/3 层在做 CNN 的事。

**预测验证**：ViT 底层 A 的熵应比文本模型同层更低（更局部），有效注意距离应随层递增（从局部→全局）——这是"被迫重建语义"的定量签名。

---

### 2. MAE（Masked Autoencoder）

```
图像 → 随机 mask 75% patch → 编码器只处理可见 patch → 轻量解码器重建被 mask 的像素
```

**S 骨架**：可见 patch 的空间结构（但 75% 被遮了——骨架比 ViT 更弱）。

**V 变异**：解码器从极少量可见 patch 重建完整图像。这给了 V 极大的自由——大量高方差特征被生成来补全像素细节。

**关键发现**（arXiv 2024）：MAE 的 patch 嵌入中 ~200 个高方差特征主导了总方差，这些特征**不包含语义信息**，只编码像素级纹理。移除这些特征后 k-NN 性能从 0.058 暴涨到 0.295（5 倍）。

**SSD 诊断**：**这是 V 污染 S 的经典案例。** 重建目标（像素级）把 V 的方差推到了极端——高方差特征占据了嵌入空间的主成分。语义骨架被纹理变异淹掉。微调能恢复表现是因为微调目标（分类）把 V 推回合理范围——不是 MAE 学到了好 S，是微调把 V 的过量压下去了。

**预测验证**：MAE 的逐层有效秩应显著高于 DINO——"虚假维度"占位。移除高方差维度后有效秩曲线应回落到接近 DINO 的水平。

**这直接验证了纯 V 上限边界**——MAE 的预训练目标（纯重建）无限接近"纯 V"端，骨架被压缩到几乎没有语义信息。

---

### 3. DINO / DINOv2（自蒸馏 + 对比学习）

```
同一图像 → 两个随机增广视图 → 教师网络和学生网络
          ↓
     两个输出应对齐（跨视图一致性）
```

**S 骨架**：**跨视图一致性强约束。** 无论你怎么裁切、旋转、变色——"这是同一只猫"的语义事实不变。DINO 的训练目标就是让 S（语义骨架）对视图扰动不变。

这是**显式训练 S 骨架**。对比 MAE——MAE 的目标是重建像素（V 最大化），DINO 的目标是跨视图语义一致（S 最大化）。

**SSD 诊断**：
- 为什么 DINO patch 嵌入无需微调就能直接做语义分割？→ S 骨架已经强到足够支撑零样本下游任务
- 为什么 DINO > DINOv2 在某些情况下？→ 数据量增加不一定增强 S——更多样化的数据引入了更多需要 V 处理的变化，如果 S/V 的架构分离不相应调整，S 可能被稀释
- 为什么 DINO 特征对图像退化最鲁棒？→ S 被训练成对增广不变——这直接增强了抗 V 噪声的能力

**预测验证**：DINO 的逐层 silhouette score（预测 #1）应高于同架构的监督 ViT——语义簇更早形成。

---

### 4. 扩散模型（Stable Diffusion, DALL-E 3）

```
文本 prompt → 编码为条件向量
随机噪声 → L 步逐步去噪（UNet / DiT）→ 最终图像
```

**这个模型修复了图像的"外层不完备"问题。**

在 SSD 中，文本 Transformer 的输出外层是完备的——softmax 在离散词表上归一化，$W_U$ 是 exact functor。图像 Transformer（如 ViT 分类头）输出的是连续 logits → 这是不完备的——没有"生成合法图像"的约束。

扩散模型把输出问题变成了一个渐进完备化过程：

```
噪声 x_T（纯 V——无 S）
    ↓ 去噪一步（从 prompt 引入 S 约束）
x_{T-1}
    ↓ ...
x_0 → 合法图像（S+V 在正确比例上）
```

**SSD 的核心解读**：扩散模型的 L 步去噪 = **一个额外的嵌套米田循环，在输出侧弥补图像的"外层不完备"**：

| | 文本 Transformer | 扩散模型 |
|---|---|---|
| **内层** | L 层 Attention+FFN | L 步去噪（UNet/DiT） |
| **S 骨架** | 残差流（强语义） | prompt 条件 + 图像先验 |
| **V 变异** | A 路由（推理选择） | 每步噪声预测（生成选择） |
| **外层完备性** | Softmax（词表归一化） | 最终步 → 合法图像分布 |
| **为什么有效** | 离散符号天然完备 | **扩散过程补了完备性**——逐步把噪声（纯 V）约束为合法图像（S+V 平衡） |

**这解释了为什么图像生成比图像理解表现更好**——生成侧扩散模型修复了外层完备性缺陷，而理解侧（ViT 分类/检测）没有对应机制。

## 二、视频模型

### 5. VideoPoet（Google，自回归路线）

```
视频 → MAGVIT-v2 分词器（3D CNN，LFQ 离散化）→ 离散 token 序列
                                                     ↓
                                          Decoder-Only LLM（自回归生成）
```

**核心操作**：用 3D CNN tokenizer 把视频压缩为离散 token，然后当文本一样喂给 LLM 做自回归。

**SSD 诊断**：

| 组件 | S/V 角色 |
|---|---|
| **MAGVIT-v2（3D CNN + LFQ）** | **预建 S 骨架！** 3D 时空卷积在 token 化时就提取了局部时空结构。这是显式的骨架预处理 |
| **自回归 LLM** | V 变异——逐 token 生成，因果 Attention 做动态路由 |
| **输出侧** | LFQ 离散化 → 类似文本的完备外层（但 token 空间远大于词表——完备性弱于文本） |

**VideoPoet 最聪明的设计**：**在进入 Transformer 前，用 3D CNN 把时空骨骼预建好。** 这直接回应了 SSD 的诊断——不要把像素级时空结构扔给 Transformer 的 V 去重算。

**VideoPoet 的致命问题**：自回归逐 token 生成长视频 → 误差累积 → 时间一致性崩塌。这是 V 在长时生成上的固有缺陷——每步生成的微小误差在 S 骨架上累积，最后骨架断裂。

---

### 6. Sora（OpenAI，扩散路线）

```
视频 → 3D VAE 压缩为连续潜变量（latent tokens）
         ↓
潜变量 → Diffusion Transformer（DiT）→ 全序列扩散去噪
         ↑
  文本 prompt 贯穿全过程
```

**SSD 诊断**：

| 组件 | S/V 角色 |
|---|---|
| **3D VAE** | 骨架预建——类似 VideoPoet 的 tokenizer，时空压缩比极高 |
| **DiT（全序列扩散）** | **外层完备性修复**——扩散过程把随机噪声（纯 V）逐步约束为合法视频 |
| **文本 prompt** | 外部 S 注入——从文本模态借骨架 |

**Sora 和 VideoPoet 的根本差异在 SSD 中的定位**：

| | VideoPoet（自回归） | Sora（扩散） |
|---|---|---|
| **骨架预建** | MAGVIT-v2（离散 token） | 3D VAE（连续潜变量） |
| **变异方式** | 逐 token 自回归（嵌套循环，单步 V） | 全序列扩散（**用扩散过程做 V**——每步全局去噪） |
| **外层完备性** | LFQ 离散化（弱完备） | **扩散过程本身 = 完备性修复**（强完备） |
| **长视频致命问题** | 误差累积 → S 断裂 | 计算量暴增（1 分钟 ≈ 100 万+ token） |
| **SSD 评价** | S 骨架强 + V 方式正确 + 外层不完备 | **S 骨架强 + 外层完备性完美 + V 计算量是瓶颈** |

> **Sora 和 VideoPoet 的差异不是"谁更好"——是它们在嵌套循环的不同位置做了正确的选择。** Sora 把完备性修复放在外层（扩散过程），VideoPoet 把骨架预处理放在前端（3D CNN tokenizer）。两者的真正分界在：扩散过程补了 VideoPoet 外层不完备的缺陷——在长视频上，Sora 的空间一致性更好是因为外层的渐进完备化更彻底。

## 三、关键洞察总结

### 1. MAE 的高方差特征 = V 污染 S 的定量证据

**这可能是框架驱动的第一个"逆向发现"**：SSD 预测了纯 V 目标（像素重建）→ 语义骨架被变异污染。MAE 那篇 2024 论文恰好确认了这一点——高方差特征在嵌入空间的主导是 V 过量的直接签名。

可进一步检验：任何重建型预训练（MAE、SimMIM、Scale-MAE）的高方差特征比例应该与它们的下游语义性能**负相关**。DINO 系列去掉了重建目标 → 高方差特征比例应显著更低。

### 2. DINO → DINOv2 → 骨架稀释假说

SSD 预测：数据量增加不一定增强 S。更多样化的数据引入了更多需要 V 处理的变异，如果架构的 S/V 分离度不变，S 的相对密度可能被稀释。DINOv2 在某些任务上弱于 DINO 是这个假说的一个信号——需要系统验证。

### 3. 视频模型的关键战场 = S 骨架预建的粒度

VideoPoet 和 Sora 都把骨架预建（3D CNN/VAE tokenizer）放在 Transformer 之前——这验证了 SSD 的论断：不要让 Transformer 的 V 去重建物理世界的时空结构。

**SSD 的独特预测**：

> 最优视频理解模型的 S 预处理（tokenizer）复杂度应该大于最优图像模型——因为视频有额外的时间骨架需要预建。同时，视频理解模型的 V（Transformer）层数应该少于图像模型的同比例——因为骨架更强、V 的负担更轻。

如果这个预测在现有模型上能成立——视频 Transformer 确实比图像 Transformer 更"浅层依赖"——那就不是猜，是框架产出的正确判断。

### 4. 扩散模型 = 在输出侧插入额外的 L' 层嵌套循环来修复完备性

| | L 层 Transformer | L' 步扩散 |
|---|---|---|
| **对应 SSD 什么** | 内层——推理 + 知识（米田循环） | 外层——完备性渐进修复 |
| **修复什么** | — | 连续模态的"输出不完备" |
| **为什么图像生成 > 理解** | — | 生成侧有扩散补完备性，理解侧没有 |

**SSD 的独特预测**：

> 图像理解模型如果也引入一个"外层的渐进完备化"机制——不是扩散去噪，而是在理解路径上做多步推理细化（类似 test-time training 或 iterative refinement）——应该能在不增加模型参数的情况下提升表现。当前图像模型的表现瓶颈在"理解是一次前向 pass"而"生成是 L' 步迭代"——两者在 SSD 中差了 L' 层循环。弥补这个差距的方向是**给理解侧加迭代推理循环**。

## 四、可检验预测（图像/视频扩展）

| # | 预测 | 检验 |
|---|---|---|
| A | ViT 底层 A 有效距离显著低于文本模型同层——前 1/3 层在做特征聚合，不是语义路由 | 逐层测 A 的有效注意距离（$\sum_j a_{ij} \cdot |i-j|$），对比 GPT-2 同层数 |
| B | MAE 的高方差特征比例与下游语义性能负相关——跨 MIM 类方法一致 | 逐方法测 200 维高方差成分的方差占比 → 与 k-NN 分类和语义分割的 rank correlation |
| C | DINO 的逐层 silhouette score 高于同架构监督 ViT——且语义簇形成更早（更低层数即出现） | 同架构（ViT-B/16）DINO vs 监督 ViT，逐层测 silhouette score，对比"半峰层数" |
| D | 视频 Transformer 有效层数（超过后增益递减的层数）低于图像 Transformer——因时间骨架分担了 V 的负担 | 同参数量视频 vs 图像 ViT，测逐层输出对最终预测的条件互信息。视频的`I(x_ℓ ; y | x_ℓ₊₁)`应更早趋于零 |
| E | 扩散步数 L' 减少 → 图像质量在某个阈值 L'_min 以下陡降——不是线性衰减 | 固定模型，减少扩散步数 50→40→...→1，测 FID/inception score 的导数，应存在斜率跃变点 |

## 五、物体槽 token：最新研究与 SSD 预测的对应

> 2024-2025 年，slot-based object-centric 方法从合成数据突围到了自然图像和视频——三篇关键论文各自实现了 SSD 架构中的独立组件。

### 1. Slot-MLLM（2025.05）——第一次把 slot attention 做进多模态大模型

Chi et al. [Slot-MLLM: Object-Centric Visual Tokenization for Multimodal LLM](https://arxiv.org/abs/2505.17726)

```
图像 → Slot Attention → K 个槽向量
         ↓
   Q-Former → 压缩
         ↓
   残差向量量化 (RVQ) → 离散 slot token
         ↓
   丢给 LLM 做 next-token prediction（理解和生成都用同一套 slot token）
```

- 关键突破：首次在**自然图像**（非合成）上实现 slot attention + 离散化
- 同时支持理解和生成（扩散解码器从 slot token 重建图像）
- **局限**：单帧——没有跨帧跟踪，物体恒存未编码

### 2. Slot-VLM（NeurIPS 2024）——物体槽 + 事件槽 = 名词 + 动词

Xu et al. [Slot-VLM: Object-Event Slots for Video-Language Modeling](https://proceedings.neurips.cc/paper_files/paper/2024/hash/0147d967a5db3b8dde08d2a327b24568-Abstract-Conference.html)

```
视频帧
  ├─→ 物体分支（高空间分辨率，低帧率）→ 物体槽 K 个 → "球、手、桌子"
  └─→ 事件分支（低空间分辨率，高帧率）→ 事件槽 M 个 → "推、弹、碎"
                ↓
         融合 → LLM → VQA 答案
```

- 和 SSD 的"物体槽 + 动作码"**几乎一一对应**
- 物体分支 = 视觉的"名词词表"，事件分支 = 视觉的"动词词表"
- **局限**：槽数 K 和 M 固定——不知道场景里有多少物体

### 3. CVPR 2025 对比槽——跨帧身份一致性

Manasyan et al. [Temporally Consistent Object-Centric Learning by Contrasting Slots](https://www.openaccess.thecvf.com/content/CVPR2025/html/Manasyan_Temporally_Consistent_Object-Centric_Learning_by_Contrasting_Slots_CVPR_2025_paper.html)

- 用对比损失强制"同一物体的槽跨帧相似"——完全自监督，不依赖运动标注
- 数学上等价于 SSD 的 track-based S 恒等路径（骨架的恒等约束）
- **局限**：中等复杂度数据集——未用于大规模视频生成

### 4. SlotAdapt（ICLR 2025）——slot + 扩散生成

Akan & Yemez. Slot-Guided Adaptation of Pre-trained Diffusion Models.

- 把 slot 表示整合进预训练扩散模型——"视频 slot"捕获物体轨迹和上下文
- **局限**：没有离散物体级别的动作码本

## 六、关键发现：拼图已在桌子上，还没人拼

| SSD 组件 | 已有论文 | 还缺什么 |
|---|---|---|
| 物体槽 token（K 个语义实体） | Slot-MLLM（自然图像 ✓） | 跨帧跟踪——Slot-MLLM 是单帧 |
| 名词/动词分离（物体槽 + 动作码） | Slot-VLM（NeurIPS ✓） | 物体槽数动态化 + 大规模生成验证 |
| 跨帧恒等路径（S 骨架不覆盖） | 对比槽 CVPR 2025（✓） | 从对比损失升级为残差流加法 + 大规模验证 |
| 扩散生成（外层完备） | SlotAdapt ICLR 2025（✓） | 缺少物体级别的离散动作码本 |
| 物理约束 S 骨架 | Seedance 2.0 STCM（✓） | 物理约束作用在 patch 上不是槽上 |
| 几何骨架 | Seedance 2.0 MM-RoPE（✓） | 几何编码作用在 patch 上不是物体上 |

> **没有一篇论文把它们全合在一起。** 合在一起 = Slot-MLLM 的离散物体 token + Slot-VLM 的名词/动词双分支 + 对比槽的跨帧恒等 + SlotAdapt 的扩散生成 + Seedance 的物理和几何约束。**这就是 SSD 预测的完整视频架构族——所有组件在 2024-2025 年被独立验证，但分散在 slot-based learning、video-language modeling、diffusion generation 三个互不引用的社区里。**

### SSD 的独特预测（视频架构方向）

> **预测 H（视频架构）**：把上述五个组件合成一个模型，物体槽级别的 token 化将带来以下可测收益：
> 1. 视频生成的长时一致性（>30 秒）显著优于同规模的 3D patch 模型——因物体恒等路径是结构保证的
> 2. 动作泛化——训练中未见过的动作组合（"推+旋转"比单独"推"或"旋转"）可通过动作码组合实现
> 3. 理解下游任务（VQA、动作定位）的零样本性能优于 patch-based 模型——因物体槽是可解释的中间表示
> 4. 序列长度从 O(帧×H×W) 降为 O(帧×物体数)，同计算预算下可处理更长视频

### 这个预测的风险

- 五个组件的工程集成复杂度可能压过理论收益——"拼起来"不是论文，是工程
- 物体槽的自动发现在自然视频上仍不够鲁棒——Slot-MLLM 的成功是静态图像，视频中遮挡和运动增加了不确定性
- 动作码本的大小（M）没有理论指导——几百到几千是猜测
