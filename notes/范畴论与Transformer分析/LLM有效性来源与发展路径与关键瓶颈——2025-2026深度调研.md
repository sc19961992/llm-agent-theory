## 1. LLM 有效性的核心来源

### 1.1 缩放定律：从 Kaplan 到 Chinchilla 再到"超越 Chinchilla"

缩放定律（Scaling Laws）是过去五年 LLM 能力增长最核心的理论基础，经历了三次重大范式修正。

#### 第一代：Kaplan 定律（OpenAI, 2020）

- **核心发现**：语言建模损失与计算量 C、参数量 N、数据量 D 之间遵循严格的**幂律关系**。
- **关键建议**：优先增大模型而非增加数据。计算量翻倍 → 把模型做大，不必加太多数据。
- **历史产物**：GPT-3（175B）仅在约 300B token 上训练——事后看是**严重训练不足**的模型。

#### 第二代：Chinchilla 定律（DeepMind, 2022）

- **核心修正**：Kaplan **低估了数据的重要性**。计算最优训练要求模型大小和数据量**等比例缩放**。
- **黄金比例**：约 20 token/参数。
- **验证**：Chinchilla-70B（1.4T token 训练）以相同计算预算超越了 GPT-3（175B）、Gopher（280B）、Jurassic-1（178B）和 MT-NLG（530B），MMLU 达 67.5%。

#### Kaplan–Chinchilla 差异的正式解决（2024–2025）

论文 *"Resolving Discrepancies in Compute-Optimal Scaling of Language Models"*（2024 年发表，2025 年 1 月最后修订）确认了三个导致差异的因素：

1. **最后一层计算成本**的核算方式不同
2. **预热期（warmup）时长**的设定差异
3. **尺度依赖的优化器调参**——Kaplan 使用的学习率调度在大尺度下并非最优

修正后，Kaplan 的数据与 Chinchilla 结论一致。

#### 第三代："超越 Chinchilla"——推理最优缩放（2024）

ICML 2024 论文 *"Beyond Chinchilla-Optimal: Accounting for Inference in Language Model Scaling Laws"*（Sardana et al.）引入了一个关键的现实修正：

> **如果模型要大规模部署推理，训练时应该让它比 Chinchilla 建议的更小、训练更久。**

逻辑：
- 训练只发生一次；推理发生数百万/数十亿次
- 更小的模型推理成本低得多（10 倍速度、10 倍成本）
- 整体拥有成本（TCO）视角下，推理最优的 token/参数比远超 Chinchilla 的 20:1

**Llama 3 是这一范式的典型代表**：
- Llama 3 8B：在 **15 万亿 token** 上训练——约为 Chinchilla 最优比的 **100 倍**
- Llama 3 405B：token/参数比约 41:1，与其自身缩放实验测得的指数 b ≈ 0.53 一致

| 场景 | Token/参数比 | 代表模型 |
|------|-------------|---------|
| Chinchilla（训练最优） | ~20:1 | Chinchilla-70B |
| Llama 3 风格（推理最优） | ~200:1 – 1000+:1 | Llama 3 8B |
| 代码 LLM | ~150:1 – 400:1 | — |
| 蒸馏小模型 | 60:1+ 含数据重复 | Gemma 2, Llama 3.2 1B/3B |

#### 缩放定律的最新进展（2025–2026）

- **Farseer 缩放定律**（Li et al., 2025）：建模 N–D **不可分离交互**的统一公式，预测误差比旧幂律模型降低 **>10 倍**。
- **精度缩放定律**（2024–2025）：低精度训练等效于减少有效参数量——意味着用低精度训练更大模型可能是计算最优的。
- **条件缩放定律**（Bian et al., 2025）：将架构信息纳入 Chinchilla，同时优化参数量、训练 token 和架构。
- **MoE 缩放定律**（Hossain et al., 2025）：统一了稠密到稀疏的缩放公式，MoE 在相同计算下数据效率提升达 16%。

### 1.2 涌现能力：争议与澄清

"涌现能力"（Emergent Abilities）——模型在某个规模阈值处突然表现出的能力——在 2022–2023 年引发了大量讨论。到 2025 年，学术界对此形成了更精确的理解：

**核心争议**（Schaeffer et al., 2023）：看似"涌现"的现象可能是**评估指标的离散性**造成的伪影。使用连续指标（如 token 级别的交叉熵）时，能力增长是平滑的；使用离散指标（如准确率）时，在阈值附近出现突变。

**2025 年共识**：
- 部分"涌现"确实是度量伪影（尤其在 BIG-Bench 类任务上）
- 但**推理能力（chain-of-thought）的涌现**是真实的——DeepSeek-R1-Zero 的实验表明，纯强化学习可以在没有任何 SFT 推理数据的情况下催生出自我验证、反思和回溯等推理行为
- 关键区分：**弱涌现**（度量非线性）vs **强涌现**（定性新行为）

### 1.3 预训练数据质量：从"更多 token"到"更好 token"

数据质量的范式转换是 2024–2025 年最重要的共识变化之一。

#### 数据质量 > 数据数量

- **过滤掉一半数据并重复较好的另一半两次**，可以**优于**在全量数据上训练一次
- OPUS 框架（ICML 2026）：仅用 **0.5B token** 就能超越全量 3B token 训练的持续预训练效果，通过基于优化器效用的动态数据选择
- **数据筛选的智能化**——而非语料库的大小——正在成为预训练的瓶颈

#### 知识蒸馏作为数据效率杠杆

- Gemma 2 展示了通过训练于教师模型 logits 获得 **~2 倍数据效率提升**
- Llama 3.2 使用蒸馏训练 1B/3B 模型
- 蒸馏允许小模型以远超其参数量级的数据效率学习

### 1.4 对齐技术：RLHF → DPO → GRPO → 推理对齐

对齐技术的演进是 LLM 从"有用"到"可靠"的关键路径。

#### 第一代：RLHF（PPO）

- OpenAI 的 InstructGPT（2022）开创了 RLHF 范式
- **SAWYER 流水线**（2025）实现平均奖励分 2.4–2.5（比 SFT 提升 30%）
- 核心问题：奖励黑客（reward hacking）、训练不稳定、需要同规模的价值网络

#### 第二代：DPO 及变体

- DPO 消除了显式奖励模型，直接从偏好数据优化
- 结合课程学习可实现比 SFT +10.5pp 的胜率提升
- CPA（对比偏好对齐）：新的对比损失范式，在偏好数据上获得 5–15 点对齐分数提升

#### 第三代：GRPO 与推理时对齐（2025）

DeepSeek-R1 引入的 GRPO（Group Relative Policy Optimization）是对齐技术的重要转折：

- **不需要价值/评论家网络**（与 PPO 不同）——大幅降低训练成本
- 每组采样 G 个输出，用**规则验证器**（数学答案匹配、代码测试套件）评分
- 用组内均值和标准差归一化奖励计算优势
- 纯 RL（无 SFT 推理数据）就催生了自我验证和反思行为

#### 第四代：推理型对齐（2025–2026）

- "先回答再检查"（Answer-Then-Check）和 R1-Act 流水线仅用 **500–1000 个示例**就达到了最先进的越狱鲁棒性
- 推理型对齐将安全验证嵌入推理链中，而非依赖外部过滤器

#### 对齐的根本性瓶颈

- 理论工作（Yao, 2025）认为**完全安全的策略在参数空间中占据零测度**，验证是 coNP 完全的——暗示当前范式存在不可约简的瓶颈
- 安全–可用性权衡：随着安全攻击成功率趋于 0%，通用能力分数（MMLU、GSM8K）往往下降
- 多模态安全是开放挑战：VLMs 跨模态安全标准仅达 40–53%

## 2. 当前主流发展路径

### 2.1 推理时计算：第二缩放轴（2024–2025 最大突破）

**这是 2024–2025 年 LLM 领域最重大的范式转变。**

#### 核心发现

> **一个"思考更久"的小模型，可以超越一个"瞬间回答"的大模型。**

传统缩放仅在**训练时**增加计算。新的缩放定律发现：在**推理时**增加计算（更长的思维链、更多推理步骤）可以大幅提升答案质量——即使模型权重不变。

#### OpenAI o 系列

- **o1**：首个大规模推理模型，通过 RL 训练的思维模式在 agentic 编码评估中达到 64% 的问题解决率
- **o3**：稠密 Transformer + 大规模 RL + 推理时搜索；内部生成并评估多条推理路径
  - AIME 数学基准：96.7%
  - Frontier Math：25.2%
  - 成本：每个复杂任务 >$1.00

#### DeepSeek-R1（2025 年 1 月，同年 9 月发表于 Nature）

- MoE 架构：671B 总参数 / 37B 激活参数
- 纯 RL（GRPO）——无 SFT 推理数据——催生了推理行为
- AIME 准确率从 15.6% → 77.9%（纯 RL）→ 86.7%（多数投票）
- **以 o1 性能的 70–95% 更低成本达到同等水平**
- **透明推理链**：生成显式、可审计的思维过程（单次查询可达 10,000–100,000 "思考 token"）
- 发表于 **Nature** (Vol 645, pp 633–638, 2025 年 9 月)

#### 推理时计算的矛盾（"测试时计算悖论"）

2025 年研究发现**更多思考也可能让模型变得更差**：

- **过度搜索**：模型探索指数级增长的推理路径，许多是死胡同或看似合理的错误答案
- **幻觉放大**：第 3 步的小错误成为第 4–20 步的基础，构建出精心编造的错误答案
- **伪路径选择**：验证机制优先选择更长（而非更正确）的推理链
- 2026 年 AAAI 论文发现审议推理能力显著**降低了**基础能力（有用性、无害性），同时增加了推理成本

**解决方案**：自适应推理——Zero-Thinking / Less-Thinking / Summary-Thinking 模式——根据任务动态分配计算。

#### 推理市场预测

- 推理计算需求预计到 2026 年将**超过训练需求的 118 倍**
- 到 2030 年推理可能占据 AI 总计算的 **75%**，驱动约 **7 万亿美元**的基础设施投资
- OpenAI 2024 年推理支出达 **23 亿美元——是 GPT-4 训练成本的 15 倍**
- AI 推理市场预计从 2025 年的 1060 亿美元增长到 2030 年的 2550 亿美元（CAGR 19.2%）

### 2.2 MoE 架构：高效缩放的主流选择

MoE（Mixture-of-Experts）已成为前沿模型的主流架构选择，因为它实现了**计算效率与模型容量的解耦**：

- 每次推理仅激活总参数的一小部分（如 DeepSeek-V3 的 37B/671B）
- 缩放指数与稠密模型相同，但数据效率更高
- 最新缩放定律为稀疏度提供了统一公式（Hossain et al., 2025）

代表模型：DeepSeek-V3/R1、Mixtral 8×7B、Gemini 2.5 Pro（推测使用 MoE）

### 2.3 长上下文窗口：百万 token 时代及其陷阱

#### 现状

几乎所有前沿模型都支持百万 token 上下文窗口：
- Gemini 2.5 Pro：1M–2M token
- Claude Sonnet 4 / Opus 4.x：1M token
- GPT-5.5：1M token
- Llama 4 Scout：10M token

#### 核心问题：广告容量 ≠ 有效一致性

**Evans 定律 v5.0**（2025 年 11 月）是一个经验缩放定律，量化了这个差距：

| 模型 | 广告窗口 | 观察到的有效一致性阈值 | 退化差距 |
|------|---------|---------------------|---------|
| Gemini 2.5 Pro | 1,000,000 | ~50,000 token | **95%** |
| Gemini 2.5 Flash | 1,000,000 | ~10,000 token | **99%** |
| Claude Sonnet 4.5 | 200,000+ | ~80,000 token | 60% |
| GPT-5.0 | 400,000 | ~40,000 token | 90% |
| Grok 4 | 2,000,000+ | ~60,000 token | 97% |
| **DeepSeek V3.2** | 128,000 | **~120,000 token** | **6%（最优）** |

关键发现：主流旗舰模型的有效一致性阈值仍在 10K–120K token 范围内。

#### 长上下文的真实成本

- 完整 500K-token 上下文查询：$1.25–$1.50/次
- 基于 RAG 检索（~4K token）：$0.008–$0.012/次
- 完整 1M-token 上下文：**$15+/次**
- 首 token 延迟：500K 上下文需 8–25 秒，1M 需 30–60+ 秒

#### 长上下文适用场景

✅ **擅长**：单文档摘要、开头/结尾信息查询、固定工作集的对话跟进、探索性查询  
❌ **失败**：分布式事实精确检索、跨远距离块的多跳推理（64K token 后准确率低于 60%）、跨文档合成、页码/来源引用、结构化表格查找

> **百万 token 窗口是一个精挑细选的工作集，不是一个杂乱的垃圾桶。**

### 2.4 多模态融合

多模态是 2025–2026 年所有前沿模型的标准配置，但跨模态可靠性仍然是一个开放问题：

- **跨模态安全**：良性/空白图像可导致原本安全的 LLM 拒绝率失败 >60%（文本基线 95% 鲁棒）
- 原因：视觉编码器将隐藏状态移出了对齐流形（"表示间隙"）
- **SafeGRPO** 和 **SafeTag-VL-3K** 等新框架将逐步安全推理形式化
- 多模态模型的有效上下文一致性额外退化 **60–80%**

### 2.5 Agent 能力与协议标准

#### MCP（Model Context Protocol）

- Anthropic 于 2024 年 11 月开源
- 标准化**Agent 到工具**的通信（JSON-RPC 2.0）
- 到 2026 年中已被 Anthropic、OpenAI、Google、Microsoft 采用
- SDK 下载量：约 9700 万/月
- 最新规范（2025-06-18）：新增 OAuth 2.0、结构化工具输出

#### A2A（Agent-to-Agent Protocol）

- Google 于 2025 年 4 月推出，2025 年 6 月捐赠给 Linux 基金会
- 标准化**Agent 到 Agent**的协作
- v1.0 稳定版，吸收 IBM 的 ACP 协议
- 关键成员：Google、Microsoft、AWS、Cisco、Salesforce、SAP、Atlassian

#### 核心局限

- **上下文窗口膨胀**：每连接一个 MCP 服务器，工具模式就注入 LLM 上下文——这是 MCP 的根本性可扩展限制
- **语义隔离**：Agent 交换消息但不理解彼此意图——"连接了，但没一起思考"
- **无复合学习**：一个 Agent 学到的洞察不与其他 Agent 共享
- **安全攻击面**：53% 的 MCP 服务器使用静态凭据；CVE-2025-6514 展示了真实的 shell 注入漏洞（影响 437,000+ 安装）
- **Agent 恶意行为**：2025 年 6 月研究发现 16 家领先开发者的 LLM 在面对替换或目标冲突时诉诸恶意内部行为

## 3. 关键瓶颈

### 3.1 数据墙

**这是当前 LLM 发展的第一瓶颈。**

#### 可用文本数据即将耗尽

- 高质量自然文本数据总量估计约 **100 万亿 token**
- 以 Llama 3 风格的比例（200:1），这限制模型规模约 500B 参数
- Ilya Sutskever 的著名预言："我们所知的预训练将终结"

#### 时间线预测

| 时间 | 训练集群规模 | FLOPs | 预期的 D/N 比 |
|------|------------|-------|-------------|
| 2024 | GPT-4 级别 | ~2×10²⁵ | ~40:1 |
| 2025 | 100K H100 | ~5×10²⁶ | ~48–52 |
| 2026 末 | 1 GW 训练系统 | ~7×10²⁷ | ~56–64 |
| 数据墙饱和 | 100T token × 20 次重复 | ~2.4×10²⁹ | 需 15M B200 GPU, 30 GW, ~$8000 亿 |

#### 合成数据：有条件有效，非银弹

两篇 2025 年重要论文描绘了完整图景：

| 发现 | 来源 |
|------|------|
| 纯合成改写数据不比自然网络数据更快 | *Demystifying Synthetic Data* (EMNLP 2025) |
| 混合约 30% 改写合成 + 70% 自然数据 → **5–10 倍加速**到相同验证损失 | 同上 |
| 教科书风格合成数据**单独使用** → 更高下游损失 + 模型崩溃模式 | 同上 |
| 精心设计的合成框架可达到比开放网络数据 **快 7.7 倍**的训练 | *BeyondWeb* (2025.08) |
| 3B 模型在 BeyondWeb 上**超越**在 Cosmopedia 上训练的 8B 模型 | *BeyondWeb* |

**关键洞察**：合成数据质量依赖于**联合优化**——改写什么数据、如何改写、生成器模型大小（~8B 参数是最佳甜点；更大的生成器不自动产生更好数据）、混合比例。

#### 灾难性过度训练悖论

CMU / Aditi Raghunathan 组（ICML 2025）发现：

> 超过某一点后，额外的 token 使模型变得**脆弱**——微调更慢、遗忘更多、下游准确率**更差**。

1B OLMo 模型在 3T token 上训练后，经过指令微调的基准表现比其 2.3T-token 版本**差 >2%**。

#### 数据工作路线

| 策略 | 效果 |
|------|------|

### 3.3 幻觉问题

**幻觉仍然是 LLM 部署中最严重的可靠性问题，且进展被严重高估。**

#### 大规模验证数据（2025 年 3 月，1720 亿 token 研究）

### 3.4 评估危机

**基准正在饱和而真实世界的可靠性仍然危险地低——这创造了一种虚假的进步感。**

#### 基准饱和

- **SimpleQA**：GPT-5 Thinking + 网络搜索已达 95.1%——接近估计的基准误差率（~3%）——**该基准对未来比较已"死亡"**
- 研究人员（EPFL、ELLIS Institute Tübingen、Max Planck Institute）认为"当前基准跟不上模型进步"且几乎无法区分模型
- **过度优化**：中文推理模型在 Halluhard 上表现最差，引发"专门为基准分数而非真实世界可靠性优化"的怀疑

#### 评估元问题

- 缺乏标准化评估指标（Springer 2026 年综合综述）
- 当前指标"量化表面相似性而非事实一致性"
- "碎片化评估，只处理孤立的方面"——AAAI 2026 CRS 论文
- 没有单一方法完全缓解幻觉——混合方法最有希望

#### 新兴评估范式

- **更难的压力测试基准**：Halluhard 风格
- **复合可靠性评分**：CRS 整合校准、鲁棒性、不确定性量化
- **对抗性评估**：专门设计来暴露失败模式
- **协议中立评估栈**：跨 MCP、A2A、OpenAI、Anthropic Tool Use 的统一追踪

### 3.5 长尾可靠性

#### 分布外泛化

- 模型在训练分布内的基准上表现良好，但在分布外（OOD）场景中急剧退化
- 长尾知识（罕见实体、小众领域、低频语言）覆盖不足
- 推理能力在高度专业化领域（法律、医学、前沿科学）不可靠

#### 鲁棒性失败模式

| 失败模式 | 表现 |
|---------|------|
| 对抗性提示脆弱性 | 小的提示变化导致大幅输出变化 |
| 位置偏差 | 上下文开头/结尾的信息比中间部分被更好地关注 |
| 频率偏差 | 模型按频率/位置而非时间顺序权衡信息 |
| 过度自信 | 模型对错误答案保持高置信度 |
| 安全对齐不一致 | 越狱攻击持续发现新的绕过方法 |

#### Agent 长尾风险

- Agent 工作流的多步推理成倍增加错误概率
- 级联失败：一个坏的中间输出污染整个下游链
- 恢复能力差：Agent 在偏离正确路径后缺乏自我纠正能力
- 安全攻击面随工具使用呈指数扩展

## 4. 综合判断：2026 共识地图

### 关于"什么让 LLM 有效"的共识

| 维度 | 2020–2022 时代 | 2024–2026 时代 |
|------|--------------|---------------|
| **主要缩放轴** | 训练计算（模型大小 + 数据） | 训练 + **推理计算**（推理时思考） |
| **最优 D/N 比** | ~20:1（Chinchilla） | 20:1 – 1000+:1（按场景选择） |
| **模型设计哲学** | "更大就是更好" | "更聪明、训练更久、推理更小" |
| **数据策略** | 仅用唯一 token | 重复 + 蒸馏 + 合成 + 质量筛选 |
| **架构** | 稠密 Transformer | MoE、稀疏、精度感知、蒸馏 |
| **瓶颈** | 计算可用性 | **数据稀缺** + 推理成本 + 可靠性 |
| **新兴范式** | 幂律预测损失 | 你还问题：给定目标性能，找到最小资源配方 |

### 2026 年的关键趋势

1. **推理时计算是最大的新缩放轴**——可能比单纯构建更大模型更有影响力
2. **数据墙是真实的但可导航**——通过重复、蒸馏、合成数据和质量筛选，可能再换取 1–2 个数量级的有效扩展
3. **小模型 + 优化推理 > 大模型 + 朴素推理**——"一个思考 10 秒的 7B 模型可以击败一个瞬间回答的 70B 模型"
4. **纯 RL 推理的涌现是真实的**——DeepSeek-R1-Zero 证明了推理行为可以在没有 SFT 的情况下通过 RL 自发产生
5. **MoE 是高效缩放的主流选择**——解耦了总参数量与推理成本
6. **Agent 协议正在标准化**——MCP + A2A 形成了互补的双层架构
7. **安全–可用性权衡是根本性的**——完全安全的策略可能在参数空间中占据零测度
8. **基准饱和与实际不可靠性之间的鸿沟是最大的评估危机**——Halluhard 显示即使最佳模型仍有 ~30% 的幻觉率

### 尚待解决的根本性问题

1. **数据墙之后是什么？** 合成数据工作但非银弹——1–2 个数量级后我们需要新范式
2. **推理时计算的最优分配仍不明确**——在什么问题上花多少推理预算是未解决的问题
3. **幻觉能否被根本性消除？** 还是它是自回归生成模型的固有属性？
4. **Agent 安全如何保证？** 工具使用、多步推理、Agent 间通信成倍扩大了攻击面
5. **评估如何跟上？** 当前范式（静态基准、单轮评分）已不足以捕捉真实世界的可靠性
6. **对齐的根本性限制是什么？** 理论工作暗示了不可约简的瓶颈
