# Token 是什么——索引论

## 唯一定义

> **Token 是 ℝⁿ 中的索引 i，是残差流的一个持久地址。其功能是承载和接收语义空间中的向量 x_i ∈ ℝᵈ，提供跨层恒常性，但不提供范畴结构——它与其他 token 的关系完全由 ℝᵈ 端的当前状态参数化。身份不自足，但正是这种不自足使语义空间能演化。**

---

## 数学定义

```
Token i = 标准基向量 e_i ∈ ℝⁿ

e₁ = (1, 0, ..., 0)
e₂ = (0, 1, ..., 0)
...
e_n = (0, 0, ..., 1)
```

**Token 是 ℝⁿ 中的位置坐标。** 非语义、非态射、非对象——就是"第几号"。

---

## Token 仅有的功能

| 功能 | 数学实现 |
|------|---------|
| **地址** | Token i 是残差流中第 i 行的地址。`X[i, :] = x_i`——在此处取出/写入向量 |
| **恒常性** | 第 1 层的 token i 和第 12 层的 token i 是同一个 i。索引穿越所有层不变 |
| **路由目标** | 注意力写入时：`x_i ← x_i + Σ_j a_{ij} v_j`——加权后的信息写到地址 i |

---

## 和 ℝᵈ 的关系

```
ℝⁿ 提供"哪里"（where）
ℝᵈ 提供"什么"（what）——路由状态向量和语义内容的混合空间

Token i：哪里   ← 索引、地址、位置，没有范畴结构
x_i：    什么   ← 路由状态向量，同时承载语义内容（由 W 编码）
                  这一向量当前同时服务于 W（语义变换）和 A（路由匹配）
                  两类用途对向量的需求不同质（高维语义 vs 低维路由），
                  混在同一个向量中是架构的欠分离——
                  深层路由退化、SAE 特征不稳定都源于此

Token i 自始至终是同一个 i。
x_i 每层都在变——被 W·A·残差·MLP 逐层重写。
```

**Token 是容器，不是内容。容器不变，内容被 ℝᵈ 端范畴态射反复变换。但当前架构中，这个"内容"是一个语义和路由的混合体——真正的类别边界在别处。**

```
    语义的全部载体：W（E、QKV投影、MLP、U）
    路由的全部载体：A（\{|i⟩\} 槽位、\{|x_i⟩\} 路由状态、\{a_{ij}\} 路由决策）
    
    |x_i⟩ 归 A 管——它是路由系统的状态向量。
    这个状态向量携带的"含义"是 W 作用的结果——就像算盘珠子的"数值"
    是算盘规则赋予的。关掉 W，|x_i⟩ 是无意义的坐标。
```

---

## 为什么不构成对象

```
范畴论对象的要求：
  ✓ 有恒等态射 id_A: A → A
  ✓ 与其他对象间有 hom 集
  ✓ hom 集支持复合

Token i 满足吗：
  id_i？    自注意力 a_{ii} 不是恒等——它做计算，且每层不同
  hom(i,j)？a_{ij} 不是态射——它在层间变化，依赖 ℝᵈ 状态
  复合？    a_{jk} 和 a_{ij} 之间没有运算得到 a_{ik}

核心障碍：
  对象要求"与所有其他对象的关系体系"是对象自身的性质。
  Token i 与 token j 的关系 a_{ij} 不是 i 的性质——是 ℝᵈ 端状态通过 Q·Kᵀ
  计算出来的。关系不属于 i，属于 (整个当前表示, W_Q, W_K) 的联合参数化。
  
  对象的身份必须自足。Token 的身份全部借自 ℝᵈ。
```

---

## 但身份的"借用" = 语义演化的引擎

token i 没有自足身份 → 它每层在 ℝᵈ 中的坐标 x_i^(l) 是"它此刻的含义"
注意力 a_{ij}^(l) 依赖此刻所有 token 的坐标 → 路由决策被语义空间状态参数化
路由决定更新方向 → 坐标变化 → 下一层坐标不同 → 路由决策也不同

**如果 token 有自足身份 → a_{ij} 固定 → 路由不变 → 语义空间静止 → 深度坍缩为 Markov 链的平稳分布。**

> **Token 是空的容器 = 它可以在不同层承载不同的含义。这个"空"不是缺陷——它是语义能够在深度上累积的前提。**

---

## 图景

```
              Token i（索引，地址——空的槽位）
              ┌──────────────────────┐
              │  e_i ∈ ℝⁿ            │  ← 不变的"槽位"
              │                      │
              │  ┌────────────────┐   │
              │  │ x_i^(1) ∈ ℝᵈ   │   │  ← 第1层的语义坐标（W 态射作用于此）
              │  └────────────────┘   │
              │  ┌────────────────┐   │
              │  │ x_i^(2) ∈ ℝᵈ   │   │  ← 第2层的语义坐标（已变）
              │  └────────────────┘   │
              │        ...            │
              │  ┌────────────────┐   │
              │  │ x_i^(L) ∈ ℝᵈ   │   │  ← 第L层的语义坐标（完全不同于第1层）
              │  └────────────────┘   │
              └──────────────────────┘

Token：同一个槽位——"谁在里面"的"在"
内容：每层都被 ℝᵈ 端范畴完全重写——"在"里面的"谁"逐层变化
关系 a_{ij}：在槽位之间路由内容，但路由规则来自内容（Q·Kᵀ）
→ 路由依赖内容 → 内容被路由更新 → 下一层内容不同 → 路由变化
→ 动力学的闭合环路
```

**Token 是"谁在里面"的"在"——不是"谁"。这个"在"的恒常性（跨层不变）和"谁"的变异性（逐层重写）的张力 = Transformer 的动力学本质。**

---

## 推论

```
Transformer 内部：
  真正的范畴对象：ℝᵈ（语义空间）——有自足身份，米田成立，W 复合成立
  非对象（索引）：{1, ..., n}（token 位置）——寄生在 ℝᵈ 上，无自足身份，米田不适用
  非对象（分量）：a_{ij}——路由决策的分量，被 ℝᵈ 当前状态参数化
  非对象（路径项）：A^(l)——参数化映射，非固定态射

因果关系：
  Token 的非对象性 → 身份借用 → 路由随语义变化 → 语义演化 ✓
  Token 若是对象 → 身份自足 → 路由固定 → 语义静止 → 同质化 ✗
```
