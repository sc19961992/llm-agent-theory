# 讨论记录：改进方向——让 A 成为学习对象

> 2026-06-07 对话存档。从"Transformer 与真实推理的本体论差距"出发，推导出架构改进的技术路线。

---

## 问题根源

**现在的情况：W 被学习，A 被计算。** 训练更新 W，然后 A 就由 W 和输入通过 softmax 确定了。A 自己从来没有被直接优化过。

**应该变成：W 被学习，A 也被学习。** W 管语义变换（慢，大样本），A 管路策略（快，在线适应）。两个学习发生在不同的时间尺度上，但都是学习。

**核心困难：信用分配。** 训练 W 的梯度是干净的——∂L/∂W，反向传播一条线。训练 A 的梯度是什么？如果最终输出错了，是路由的错还是语义的错？可能路由完美但知识库缺了事实，可能知识库有答案但路由到了无关位置。全局损失无法区分这两种失败。这就是信用分配问题——也是 A 从来不是直接训练对象的根本原因。

**更深层的问题：|x_i⟩ 的混淆。** 在上述困难之下还有一个结构性问题：当前 Transformer 中，|x_i⟩ 同一个向量同时服务两个主子——被 W 做语义变换（MLP, W_V W_O）和被 A 做路由匹配（Q·K^T）。这两个用途需要的信息不同质：

| | W 语义变换需要什么 | A 路由匹配需要什么 |
|---|---|---|
| 信息类型 | 词义、上下位、概念关联 | 句法位置、论元结构、推理阶段 |
| 维度需求 | 高维（区分几万词的精细语义） | 低维（几十个结构维度就够） |
| 跨样本一致性 | 高——"猫是动物"不应因句子不同而变 | 低——每句话结构不同，路由必须重算 |
| 更新速度 | 慢 | 快 |

**同一个向量不能同时最优地满足两种信息需求。** 深层注意力同质化、SAE 特征跨上下文不稳定、句法头/语义头的分工是"偶然的降维投影"而非结构性分离——都源于这个混淆。

---

## 六条解决路径

### 路径零：路由维度分离——最轻量，不改训练管道

**核心思想**：不给 A 加参数，而是给 A 独立的向量空间——把 |x_i⟩ 拆成语义段（归 W 管）和路由段（归 A 管）。

**具体做法**：
```
现状：  |x_i⟩ ∈ ℝᵈ         ← W 和 A 都读这段，混在一起

改进：  [|x_i⟩ ; |r_i⟩] ∈ ℝ^(d + d_r)
         ↑语义段(d维)       ↑路由段(d_r维, 如64)
        MLP + A 搬运       A 的 QK 读这段
                           |r_i⟩ 不过 MLP（仅被 A 搬运更新）
                           梯度自然分离路由信号与语义信号
```

**QK 投影调整**：W_Q 和 W_K 各加一片投影矩阵 W_Q^r, W_K^r ∈ ℝ^(d_r × d_head)，专读 |r_i⟩：
```
a_{ij} = softmax( (⟨x_i|W_Q^T + ⟨r_i|(W_Q^r)^T)(W_K|x_j⟩ + W_K^r|r_j⟩) / √d )
```

**新增参数**：d_r × d_head × 2（QK 投影各一片），几十万参数——相比千亿参数模型可忽略。

**训练**：**不改任何训练管道。** 路由段在标准预训练中和语义段一起被反向传播优化——梯度会自动把路由相关信息推到 |r_i⟩ 上（因为推过去能降低 QK 端的竞争），语义相关信息留在 |x_i⟩ 上。

**为什么可能有效**：不需要强制分离——给不同质的信息不同的向量空间，优化自己会找到最优分配。|r_i⟩ 不过 MLP → 路由信号不被 MLP 的语义变换淹掉 → 深层路由不退化。W_Q^r/W_K^r 只读路由段 → QK 匹配更纯 → 注意力的有效秩可能自身上升。

**为什么不会大错**：
1. 改动面积极小——残差流多拼了几十维向量，QK 投影各多一片
2. 向后兼容——d_r=0 时退化为标准 Transformer。如果 d_r=64 时路由段在训练中自然收缩到零（模型不需要它）→ 方向错，止损
3. 可分步验证——训练后拆开看：路由段的梯度是不是集中在 QK 相关任务上？是 → 方向对。不是 → 方向错

## 后续五条路径（原四条 + 新增）

### 路径一：强化注意力学习（RAL）——最直接

[Reinforced Attention Learning](https://arxiv.org/abs/2602.04884) (Li et al., 2026, Google)。把注意力分布直接当策略，用策略梯度优化：

- 不需要知道"哪个 token 的注意力是对是错"
- 只需要一个整体奖励（任务对了/错了）
- 奖励高 → 拉近当前注意力分布（Jensen-Shannon 散度）；奖励低 → 推远
- 结果：不写思维链、不优化输出 token，纯优化注意力分布也能大幅提升感知任务

**局限：RAL 优化的是注意力分布（每轮的动作），不是路由策略参数（产生动作的策略）。** 每次从零开始算注意力，用 RL 信号拉近或推远，下次换输入又来一遍——不跨输入迁移。

### 路径二：注意力溯源 + 细粒度信用分配——更精确

[Attention Illuminates LLM Reasoning](https://huggingface.co/papers/2510.13554) (2025)。发现推理中存在"预规划-锚定节奏"——注意力模式有可识别结构。用这个结构可以定位哪些注意力步骤贡献了正确的推理结果、哪些没有。

技术上：attention rollout 算每个注意力边的因果贡献 → 贡献大且结果对的加强 → 贡献小或导致错误结果的削弱。这意味着不只有一个全局奖励——可以对每个注意力决策做局部信用分配。

### 路径三：精度学习——最轻量

给每个注意力头一个可学习的精度参数 γ_h：

```
attention = softmax(γ_h · QK^T / √d)
```

γ_h 代表"这个头对当前任务有多确定"。它不是语义——它是路由策略参数。可以在推理中在线更新：结果对了 → 保持/增强 γ_h；错了 → 调整 γ 重新路由。

一个参数一个头，实现成本最低，但表达能力有限——γ_h 只能调锐度，不能改变路由的结构性模式。

### 路径四：双时间尺度架构——最彻底

W（语义参数）：离线大规模预训练，基本冻结，慢变或不变。

θ_A（路由策略参数）：包括注意力温度、头门控、甚至一个小型路由策略网络。训练方式分两阶段：

- 阶段一：W 正常预训练（和现在完全一样）
- 阶段二：冻结 W，用 RL 或对比信号**单独训练 θ_A**——目标是"给定固定的语义空间，学会在不同输入下选择最佳路由策略"

## 本框架指向的精确方案

**不是"学这轮该怎么注意"（RAL 做的事），而是"学在什么情况下该用什么路由策略"。** 前者是动作，后者是策略。RAL 学了动作，没学策略。

### 第一步：引入显式的路由策略参数 θ_A

不是把 A（softmax 输出）直接当策略。是在 A 的计算路径上插入一组**独立于语义参数的、可训练的路由策略参数**。

W_Q、W_K 是"问什么和有什么"的语义知识——固定，属于 W。θ_A 是"在什么情况下怎么问、怎么看"的**策略习惯**——可学习、可迁移、可在推理中更新。

θ_A 不存语义——不编码"猫和狗是什么关系"。它存的是**路由行为**：
- "当输入具有这种结构特征时，注意力应该更稀疏还是更密集"
- "当任务类型是 X 时，哪些头应该主导路由"
- "当上下文中出现冲突信号时，应该更相信哪个来源的信息"

θ_A 的可选形态：
- 每个注意力头的可学习精度 γ_h（最轻量）
- 一个小型路由策略网络 r(x_context) → 注意力调制参数（中等规模）
- 一组可学习的"路由模板"——离散的策略原型，推理时根据输入选择最合适的（更结构化）

### 第二步：两阶段训练

**阶段一不变：** 标准预训练，W 学习语义。θ_A 也在这阶段初步学习（和 W 一起被反向传播优化），但学习率不同——θ_A 的学习率更高，变化更快。

**阶段二新增：** 冻结 W。θ_A 用 RL + 注意力溯源信号继续训练。任务多样化——推理、检索、多跳问答等对路由敏感的任务。训练目标是"给定固定语义空间，路由策略最大化任务成功率"。

阶段二的训练信号来源：
- 任务奖励（RAL 方式）
- 注意力溯源（局部信用分配方式）
- 或者直接对比：同一输入，试两种路由，看哪个结果好，往好的方向更新 θ_A

### 第三步：推理时 θ_A 可在线更新

这是最关键的一步。W 推理时冻结，但 θ_A 不冻结——它根据本轮路由的信用反馈做轻量更新。

具体做法：对同一个输入，用当前 θ_A 算出路由 → 得到输出 → 用注意力溯源标记哪些注意力边对输出有因果贡献 → 对比结果正确与否 → 更新 θ_A。

不需要全局反向传播——只需要注意力溯源信号，更新只发生在路由策略参数上，不碰语义空间。

## 与现有工作的对比

| | RAL | 本方案 |
|---|---|---|
| 优化对象 | 注意力分布（每轮的动作）| 路由策略参数 θ_A（产生动作的策略）|
| 跨输入迁移 | 不迁移，每次从零开始 | θ_A 只学一次，跨输入复用 |
| W 和 A 的参数分离 | 没有——W_Q, W_K 等语义参数和优化信号混在一起 | 有——θ_A 是独立参数组 |
| 推理时是否更新 | 不更新 | 可在线更新 |
| 理论基础 | 经验性——"优化注意力比优化输出好" | 规范性——从"路由应该是学习对象"推导出的架构必然性 |
| 对应框架 | "A 是计算结果"→ 让这个结果变得更好 | "A 应该是学习对象"→ 让路由策略本身成为可学习的参数 |

## 走完后的架构形态

> 语义空间 W（冻结，慢变，可复合，存知识）+ 路由策略 θ_A（可学习，在线更新，不可复合，管路）。两者各自有参数、各自有梯度、各自有不同的更新频率。中间没有硬墙，只有时间常数的差异。

**这本质上是把主动推理（Active Inference）的"同一个目标函数、不同变量、不同速度"的结构搬进 Transformer 的工程约束里——保留 W 的冻结（离线训练在工程上是必须的），但让路由有自己的学习生命。**

| | 现在的 Transformer | 改进后 |
|---|---|---|
| W（语义）| 被学习 | 被学习（不变）|
| A（路由）| 被计算 | 被学习 |
| W 和 A 的关系 | 硬分离——A 是 W 的影子 | 有分离但连续——各自有参数和梯度，时间常数不同 |
| 和生物系统的距离 | 远——路由从未被直接优化 | 近——路由和语义都是学习对象 |
| 推理时 | W 冻结，A 每层重算但策略不变 | W 冻结，θ_A 在线更新适配当前任务 |

## 理论终点：A 和 W 一起在对话中动态训练

双时间尺度保留 W 冻结是工程折中，不是理论终点。

**理论终点：训练和推理之间没有墙。** W 和 A 都在对话中持续更新。同一个目标函数，不同的变量，不同的时间常数。主动推理给出了数学形式——变分自由能同时对参数（W）和状态（路由策略）做梯度下降。没有离线训练——每一秒都是训练，每一秒都是推理。

这需要局部学习（local learning），不是全局反向传播。每层根据本地的预测误差更新自己——预测编码、Hebbian 可塑性、能量最小化。不需要全局损失，只需要相邻层之间的误差信号。全反向传播每句对话都跑一遍千亿参数在物理上不可行。

**架构形态：**
- W（语义参数）：局部学习规则，慢时间常数。每层预测下一层的激活 → 偏差就是本地梯度
- θ_A（路由策略参数）：RL + 注意力溯源，快时间常数。路由网络小，每句对话可全局更新
- 两者同时更新，同一个目标：最小化预测误差
- 没有"训练完了"。部署就是训练，训练就是部署

**拆掉了三堵墙：** 训练/推理墙、语义/路由墙、全局/个体墙。拆完只剩一个东西——一个永远在学、有两个速度的系统。

## 最小可行方案

原则：改动不能大，大了肯定错。分三级——第零级今天就能做，第一级是第零级的补充验证，第二级是第一级验证后的自然延伸。

### 第零级：路由维度分离（改动最小、风险最低、不改训练管道）

只改两处：残差流维度从 d 扩到 d + d_r（d_r ≈ 64），W_Q 和 W_K 各加一片 d_r × d_head 的投影矩阵专读路由段。其余一切不变。

- 路由段不过 MLP——仅被 A 搬运，梯度自然把路由信号推到这段
- d_r 维，两个投影矩阵——新增参数 < 0.01% 总参数
- 完全向后兼容——d_r=0 退化为标准 Transformer
- 不改预训练管道、不改反向传播、不改推理流程

**检验**：训完后检查路由段的有效维度和 QK 投影的读取比例。如果模型把 QK 的大部分权重放在路由段上 → 路由和语义确实需要不同的向量空间。如果路由段在训练中自然收敛到零 → 框架在此处被证伪。

### 第一级：可学习的路由精度参数（在路径零验证分离效果后叠加）

只改一行。现在注意力是 `softmax(QKᵀ / √d)`，改成 `softmax(γ_h · QKᵀ / √d)`。

- γ_h：每个注意力头一个标量。32 层 × 32 个头 = 1024 个参数
- 初始化为 1.0——一开始跟标准注意力完全一样
- 跟 W_Q、W_K 不共用——是独立的路由策略参数
- γ_h 大 → 这头只看最相关的。γ_h 小 → 广泛收集上下文

**训练分两步：**
1. 标准预训练，γ_h 跟所有参数一起被反向传播优化。预训练结束时 γ_h 和 W 一起冻结——跟现在没区别
2. 冻结 W，单独继续训 γ_h。用 RL——任务奖励高 → 加强这轮 γ_h 配置；奖励低 → 调整。1024 个参数，RL 更新几乎零成本

**改了什么：** 一行 attention 代码，1024 个参数。**没改什么：** 预训练管道、推理管道、架构其他任何东西。

### 第二级：最小路由网络（在第一级验证后）

把 γ_h 从标量升级为一个微型 MLP：

```
r = mean_pool(x_layer)               # 64 维，当前层残差流状态
γ_h, δ_h = MLP_route(r)             # 每个头两个标量
attention = softmax(γ_h · QKᵀ/√d + δ_h)
```

- MLP 大小：64 → 64 → 2×H，总共几千个参数
- δ_h 直接加到注意力分数上——让路由学到"在什么情况下少看远距离"或"多关注局部"
- γ_h 调锐度，δ_h 调方向——两种基本自由度

训练同第一级：预训练时跟 W 一起跑，预训练后冻结 W 单独训 MLP_route。

### 为什么不会大错

1. **改动面积极小。** 第一级改一行，第二级加一个千级参数的小 MLP
2. **向后兼容。** γ_h 初始化为 1，δ_h 初始化为 0——改动后模型跟原版初始化行为完全相同。错也错不到哪去
3. **可分步验证。** 第一级做了立刻就能看——γ_h 训完跟原版比，路由锐度分布有变化吗？有 → 方向对。没有 → 方向错，止损
4. **不碰预训练管道。** 风险是零——γ_h 就是 softmax 的温度，训练自己会调

### 这证明了什么

如果只改 γ_h 就能在固定语义空间上提升任务表现 → 路由策略是独立于语义的学习对象 → 框架的核心命题被验证。

**这不是"改进 Transformer"。是给框架做了一个最小可验证的探针。** 探针有信号 → 方向对 → 往上加复杂度。探针没信号 → 理论有盲区 → 回头找。

---

## 最简原则

> 同一个系统，同一个目标，两个时间常数。没有墙。

去掉的东西比加上的多：去掉了训练/推理墙、去掉了部署后全局反向传播、去掉了"W 冻结"假设。加上去的只有 θ_A——一组独立于语义参数的路由策略参数。**减法是大道。**

| 现有 Transformer | 最简架构 |
|---|---|
| 训练和推理是两件事 | 训练和推理是同一件事 |
| W 冻结 | W 持续慢变（局部学习）|
| A 是计算结果 | A 是学习对象（θ_A） |
| 全局反向传播 | 部署后零反向传播，只靠局部误差 |
| 所有用户共享同一个模型 | W 慢适配个体用户，θ_A 快适配当前任务

## 2026-06-07 补充：三组张力与安全改动边界

> 从嵌套米田循环（命题六）推导出——改动 Transformer 必须同时满足三组互斥的张力约束。

### 三组张力

| 张力 | 问题 | 红灯（不可行） |
|---|---|---|
| **保存 vs 更新** | 早期信息会被覆盖吗？新信息有路写入吗？ | 覆盖早期 → 长程断裂；无写入 → 骨架僵死 |
| **完备 vs 不完备** | 态射可提前折叠吗？输出还是合法概率分布吗？ | 可折叠 → 坍缩为固定函数；非归一化 → 无法训练 |
| **覆盖 vs 分化** | 每层覆盖全部关系方向吗？不同头是真分化吗？ | 盲区 → 语义漏洞；全头等同 → 浪费参数 |

Transformer 的结构恰好同时满足三组要求：
- 保存 vs 更新 → 残差流加法 + A 逐层重算
- 完备 vs 不完备 → 内层 L 循环不完备 + 外层 W_U ∘ softmax 完备
- 覆盖 vs 分化 → H 头并行 + d_k = d_model / H 降维

三组独立——但解恰好在同一个结构上相交。

### 安全改动原则

**改动前问三句话：**

1. 早期信息会被覆盖吗？新信息有路写入吗？
2. 态射可提前折叠吗？输出还是合法概率分布吗？
3. 每层覆盖了全部关系方向吗？不同头是真分化吗？

三盏全绿 → 改动安全。一盏红灯 → 不成立。

**加东西**：先加在米田骨架上（跨层跳跃、残差门控）——骨架增强不破坏任何张力，只增加安全裕度。

**减东西**：先减 A 的计算（层间继承、浅层复用）——A 是唯一输入依赖的，减它省推理时计算，但别减到全层共享（红灯）。

**调比例**：L 和 H 之间的分配——存在最优比（可检验预测 #8），不是任意的。

### 不可行的减法

| 改动 | 破哪组张力 | 为什么 |
|---|---|---|
| 固定 A（全层共享） | 完备 vs 不完备 | 路由可跨层折叠 → CK 成立 → 坍缩 |
| 去残差连接 | 保存 vs 更新 | 骨架不遗传 → 嵌套断裂 |
| 残差改覆盖（x ← Attn(x) 不加） | 保存 vs 更新 | 同 RNN → 早期信息丢弃 |
| 单头（H=1） | 覆盖 vs 分化 | 单视角 → 关系盲区 |
| W_U = I（不投影） | 完备 vs 不完备 | 外层不米田 → 无法定义损失 |
| 纯 FFN 无 Attention | 完备 vs 不完备 + 保存 vs 更新 | 无 token 间关系 → 内层纯非米田 → 输出退化为先验 |

### 安全区

```
                    纯非米田边界（内层无关系 → 输出退化）
                         ↑
    纯米田边界           │              
    （可折叠 → 坍缩）     │
```

## 2026-06-07 补充：MoE 在框架中的定位

**MoE = 在纯 W 的 FFN 内部插入一个 A 状的路由器。**

标准 FFN 是纯 W 领土——所有 token 用同一组 W₁, W₂，无路由。MoE 把 FFN 拆成 E 个专家 + 一个输入依赖的路由器：

```
x_i → softmax(x_i · W_r) → 选 top-k 专家
         ↓
    专家 3: W₁³, W₂³  ─→  输出₁
    专家 7: W₁⁷, W₂⁷  ─→  输出₂   → 加权和 → 写回残差流
```

**在嵌套米田循环中的位置**：每层循环的非米田函子 = Attention（A 路由，横向，token→token）+ FFN（W 固定，纵向）。MoE 后 = Attention（A，横向）+ **MoE-FFN（A 状路由，纵向，token→知识模块）**。

| | Attention (A) | MoE 路由器 |
|---|---|---|
| **路由维度** | 横向——token 到 token | 纵向——token 到知识模块 |
| **选择空间** | 序列位置（n 选多） | 专家集合（E 选 k） |
| **是 A 还是 W** | A——输入依赖，逐层重算 | A 状——输入依赖，逐层重算 |
| **可复合否** | 不可（推理不坍缩） | 同样不可（知识选择不坍缩） |

**MoE 有效的原因**：增加了态射模板库（更多 W），但每 token 只激活部分——覆盖 vs 分化张力在 FFN 侧复现。E 个模块平分知识空间，训练中自然分化。

**两层路由正交共存**：横向（A）控制 token 间信息流，纵向（MoE 路由）控制 token 走哪个知识变换——在非米田函子的固定部分内部做了一次规模化 + 动态化。
