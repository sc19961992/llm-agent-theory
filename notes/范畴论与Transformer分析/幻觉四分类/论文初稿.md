# 大语言模型系统性幻觉的四分类框架：机制、近似正交与诊断路由

## 摘要

当前LLM幻觉研究缺乏基于失效机制的分类体系。主流的事实性/忠实性二分停留在现象层，数据/训练/推理三阶段分类停留在成因层，二者均混淆了不同层级的分析单位。本文在任务被良好定义且排除训练诱导偏差的前提下，将系统性幻觉按失效机制分为四种：误差传播（解码阶段token级采样偏差的自回归放大）、路径缺失（推理程序在训练数据中覆盖不足）、容量溢出（信息跨度超出注意力的有效覆盖范围）和注意力漂移（注意力路由失准，在容量范围内看错位置）。本文论证该四分类的近似正交性（以主导失效模式为归因标准，边界混叠有限）和在给定边界条件下的近似完备性。基于此提出最小成本诊断树——以递增成本的简单干预（重试→拆解→缩短→锚定）替代内部指标监测，诊断与修复同步完成。附录给出可直接嵌入Agent系统提示的幻觉自检Skill。本文为纯概念框架论文，实证验证留待后续工作。

**关键词**：LLM幻觉；误差传播；推理路径缺失；有效上下文；注意力漂移；近似正交；诊断树

## 1. 引言

大语言模型的幻觉问题已经积累了大量的分类、测量和缓解工作，但缺乏一个同时满足两个条件的分类体系：(a) 每个类别对应明确的失效机制，而非仅描述现象或罗列成因；(b) 类别之间大致可区分（近似正交），整体覆盖推理全链条的主要失效模式（近似完备）。

Huang et al. (2025, ACM TOIS) 的综述代表了当前的主流范式——将幻觉分为事实性幻觉和忠实性幻觉。这一二分法在现象层是完整的，但在机制层则不然：多个不同的失效机制可以产生"事实性幻觉"这一相同现象。另一方面，按数据→训练→推理三阶段划分的分类抓住了因果链条，但混淆了"根因"和"失效机制"——训练数据中的长尾分布是根因，但它在推理时以什么机制产生幻觉，才是分类需要回答的问题。

**本文的核心主张**：在任务目标和前提被良好定义的前提下，LLM的系统性幻觉可按失效阶段和机制划分为四种类型。以此为底座，可构建一个任何用户都能立即使用的诊断树——不以模型内部指标诊断，而以递增成本的简单干预试探诊断。哪一步恢复了正确输出，即同时完成了类型判定和问题修复。

值得注意的是，这四种类型恰好对应了Transformer的三个核心功能层——FFN知识存储、注意力上下文路由、自回归解码——暗示了失效的结构性而非偶然性。这一线索将在讨论部分重新拾起。

**方法论定位**：本文为概念框架论文，目标是提出一个机制清晰的分类体系并论证其内部一致性，而非在特定基准上报告分类准确率或诊断树成功率。框架的可检验预测和实证验证方案留待后续工作。

## 3. 四分类详述

### 3.1 类型1：误差传播

**定义**：自回归解码中，某步从条件概率分布中偶然采样到一个低概率token。该偏差token进入后续所有步的隐藏状态计算，被自回归机制逐层放大为整段偏离。这不是"不知道"——是"恰好输出错了"，但后文被这个错误引导偏离。

**数学基础**：自回归分解 \(P(x_{1:T}) = \prod_{t=1}^{T} P(x_t | x_{1:t-1})\) 隐含了一个不对称：每一步的输入是前一步的输出，而非优化目标。第k步的采样偏差ε_k使后续条件分布系统性偏离——\(P(x_{k+1} | x_{1:k} + \epsilon_k) \neq P(x_{k+1} | x_{1:k}^*)\)。Arora et al. (2022)给出误差累积上界T²ε。

逐token贪心最优不等于序列级全局最优：即使在每个位置选中条件概率最大的token，联合概率也可能远低于全局最优。换言之，该机制是**传播/放大**机制，而非偏差的起源。

**类比**：笔误。不是不会写，只是恰好写错了。但如果后文围绕这个错字展开，就可能整段跑偏。

**经验证据**：Mekouar (2024)[^1]在多模型上验证幻觉token的正自相关（lag-1显著），温度放大该效应。Zhang et al. (2024)发现GPT-4能识别87%的自身幻觉，但早期幻觉一旦产生，后续错误不可避免。

### 3.2 类型2：路径缺失

**定义**：模型权重中同时存储语义知识（事实、概念关联，单步前向传播即可提取）和推理路径（多步隐含展开的思维链）。语义知识因训练数据覆盖面广而相对充足，但推理路径在训练数据中密度低得多。若某条推理路径覆盖不足或完全缺失，模型无正确程序可执行。

**关键区分**：这不是"不知道事实"——模型可能拥有所有相关事实。这是"不知道怎么把事实串成推理"。

**一个操作性判据**：若将任务拆解为清楚的子步骤提供给模型（"先做A，再做B，然后综合A和B给出C"），模型能够正确执行每个子步骤并给出最终答案，则问题确为路径缺失（子能力存在但不会编排）。若拆解后仍然错误，则可能是事实知识本身缺失或权重中编码了错误信息，归入"超出四分类范围"。

**内部机制**：FFN/MLP层存储可跨样本泛化的语义知识（Geva et al., 2021; Meng et al., 2022），自注意力机制每层动态编排信息流向。多步推理需要L层中注意力路由的特定序列编排——若某推理路径对应的路由模式从未在训练数据中出现，即使FFN存储了所有需要的子知识，注意力也无法正确编排（Brinkmann et al., 2024）。

**类比**：学生知道所有公式定理，但没学过这类题型的解题步骤。

**经验证据**：Reversal Curse（Berglund et al., 2023）——GPT-4正向79%、反向33%。Nezhurina et al. (2024)的AIW问题——顶尖模型在简单组合推理上崩溃至~0%。Li et al. (2024)发现程序性知识仅在简单逻辑任务中被有效利用。

### 3.3 类型3：容量溢出

**定义**：**有效上下文**（注意力实际能高质量聚焦的token范围，随模型和任务类型变化——从~100K到~1M不等）远小于**最大上下文**（架构层面可接受的token上限，可达1M+）。当任务所需信息跨度超过有效上下文时，关键信息超过注意力的覆盖能力。

**有效上下文 vs 最大上下文**：最大上下文由位置编码和KV缓存的硬约束决定。有效上下文由注意力稀释（Gu, 2026给出形式化上界O(1/L)）、位置衰减（Chen et al., 2026用平均场理论推导临界缩放律βₙ ≍ log n）等软约束决定。Paulsen (2025)实测发现，部分模型的有效上下文可低至标称值的1%，但最新代模型（Gemini 2.5 Flash, GPT-4.1）的有效/标称比已接近98%——差距因模型而异，非固定值。

**类比**：一本1000页的书，你能有效精读的可能只有100页。中间900页只是翻过。

**经验证据**：Hsieh et al. (2024)发现U形注意力曲线——中间位置信息系统性低权重。Ebrahimzadeh & Salili (2026)发现更长上下文在信息稀释时反而降低准确率。

### 3.4 类型4：注意力漂移

**定义**：即使信息在有效上下文范围内，注意力也未必落在正确的位置。自注意力没有内建的"应该看哪"的保证——它受位置偏差（首因/近因效应）、内容相似度干扰（语义相近但不相关的文本块在注意力竞争中胜出）、以及缺乏显式检索路由的影响，可能聚焦于无关区域而遗漏关键上下文。

**与类型3的关键区别**：类型3是"装不下"（信息量超过了注意力容量）。类型4是"走神了"（信息在容量内，但注意力路由到了别处）。在实践中，两者的区分可通过干预反馈实现（见§4和§6）：分块有效→类型3主导；分块无效但锚定有效→类型4主导；两者皆有效→优先归入类型3（锚定的改善可能为分块后的附带效应）。

**类比**：还是那本1000页的书。你只读了100页，这100页都在有效容量内，但你没看目录，或者被一段好看的比喻吸引了注意力——读完了但不知道主旨。

**经验证据**：Liu et al. (2024)将注意力偏差作为经验现象发现并命名。Chuang et al. (2024)发现注意力回看率下降是上下文幻觉的实时预测信号。An et al. (2024)通过信息密集型长上下文训练针对性修复注意力覆盖。

### 3.5 总览

| # | 类型 | 失效阶段 | 对应架构组件 | 失效机制 | 类比 |
|---|------|---------|------------|---------|------|
| 1 | 误差传播 | 解码生成 | 自回归解码层 | 采样偏差自回归级联放大 | 笔误 |
| 2 | 路径缺失 | 权重编码 | FFN知识存储层 | 推理程序训练覆盖不足 | 没学过 |
| 3 | 容量溢出 | 上下文利用 | 注意力路由层 | 信息跨度>注意力覆盖能力 | 装不下 |
| 4 | 注意力漂移 | 上下文利用 | 注意力路由层 | 注意力路由失准 | 走神 |

## 4. 近似正交性论证

四个类型的归因标准基于**可操作的干预响应**——以下是判归逻辑。

**判归类型1**：模型具备任务所需的知识和上下文（非类型2-4），但在某解码步偶然采样到低概率token。可区分性：重新生成（不做任何修改）后以较高概率回到正确轨道。

**判归类型2**：相关信息已在上下文中且注意力正确覆盖（非类型3、4），但模型在从信息到结论的推理步骤中存在系统性错误——该错误在多次独立采样中一致出现（排除类型1的偶然性）。可区分性：提供分步子步骤（chain-of-thought）后可在某些情况下修复——子能力存在但不会编排。

**判归类型3**：使用上下文压缩/分块策略后性能显著提升。分块有效→容量溢出。若分块和锚定同时有效，优先归入类型3（锚定的改善可能是分块缩小搜索空间后的附带效应）。

**判归类型4**：信息在有效容量内（分块无效→排除类型3），模型具备推理能力（拆解无效→排除类型2），但一致性地忽略某些关键上下文区域（重试无效→排除类型1）。可区分性：显式标记或重新定位关键信息后显著改善。

### 4.1 正交性限制

上述归因标准在实践中是近似可操作的，但存在三种混叠边界：

1. **类型1↔类型2的渐进边界**：当不一致性处于"单次偶然"和"多次系统性"之间时，归因取决于采样次数K和一致性阈值。这是一个统计检验的分辨率问题，而非分类的概念缺陷。

2. **类型3↔类型4的区分依赖有效上下文的知识**：若事先不知道模型的MECW，分块有效本身既可能意味着"信息超过了真实容量"（类型3）也可能意味着"分块缩小了搜索空间从而间接修复了注意力漂移"（类型4）。本框架采用实用优先规则——分块有效→归入类型3；分块无效但锚定有效→归入类型4。

3. **"主导失效模式"归因依赖于诊断树的试探顺序**：诊断树按成本递增排列干预——这意味着类型判定是有序的（重试优先于拆解优先于分块优先于锚定）。这是实用主义的判定（"什么干预最便宜且最先有效"），而非机制层面的独立归因。两者不应混淆——本文在§4中讨论的近似正交性基于机制分析，在§6中讨论的诊断树基于实用主义判定。两者一致但不恒等。

## 5. 完备性论证

### 5.1 边界条件

- 任务目标和前提被良好定义：排除对齐诱导幻觉（RLHF迎合、不确定性压制）
- 排除训练数据投毒、错误标注等上游根因（它们是诱发条件，非失效机制本身）
- 聚焦推理时系统性失效，不覆盖对抗攻击、分布外漂移

### 5.2 覆盖范围

四种类型覆盖模型推理的全部三个关键阶段和对应的架构组件（§3.5表）。

### 5.3 可能遗漏的类型

**组合组装失败**（子路径存在但组合为新推理链时出错）：若拆解为子步骤后模型能正确执行每步并综合出最终答案→归入类型2（路径缺失——路径存在但不会编排）。若拆解后仍不能→超出了本文"任务良好定义"和"排除训练根因"的双重边界（可能是权重中编码了错误知识）→属归因精度问题而非分类覆盖问题。

**校准性解码错误**（logits本身指向错误方向而非小概率token）：若一致性错误→根源在权重→归入类型2。若偶然性→归入类型1——类型1不要求token必须是低概率，只要求偏差在传播/放大机制下被级联夸大。

### 5.4 完备性结论

在给定边界条件下，四分类覆盖了推理全链条的系统性失效模式。超出此范围的案例（权重知识错误、对齐诱导偏差、训练数据投毒）不属于本文对"系统性幻觉"的操作性定义——它们是不可或缺的重要问题，但在本文的"任务良好定义"边界外部。分类的操作定义——即每一类对应一个特定的干预反应模式——使得它可以被直接转化为递增成本的诊断树。这是下一节的内容。

## 6. 诊断树：最小成本干预

四分类的直接应用不是构建复杂的内部状态监测系统，而是一个任何用户都能立即使用的诊断树。核心原则：

> **不以内部指标诊断，以干预反馈诊断。哪个手段有效，就暴露了真实的幻觉类型。**

**前提说明**：诊断树假设存在一个可执行正确性验证的机制——人工检查、基于规则的校验或独立验证模型。验证本身不在本文讨论范围内。

### 6.1 四步递增成本试探

```
发现幻觉
  │
  ├─ 第一步：重新生成（成本：1次推理）
  │   同一输入再问一次。
  │   ├─ 恢复 → 类型1（误差传播）。完成。
  │   └─ 仍错 → 继续。
  │
  ├─ 第二步：拆解步骤（成本：添加"请逐步推理"）
  │   ├─ 恢复 → 类型2（路径缺失）。CoT将长程推理拆为短程子步骤。
  │   └─ 仍错 → 继续。
  │
  ├─ 第三步：分块处理（成本：截断或分块上下文 + 1至N次推理）
  │   丢弃中间段落只保留首尾，或分块独立处理后聚合。
  │   ├─ 恢复 → 类型3（容量溢出）。分块后每段在有效范围内。
  │   └─ 仍错 → 继续。
  │
  └─ 第四步：锚定关键信息（成本：修改提示词 + 1次推理）
      ├─ 恢复 → 类型4（注意力漂移）。信息一直在，但模型在看别处。
      └─ 仍错 → 超出四分类范围
```

### 6.2 顺序逻辑

| 步骤 | 成本 | 为什么排在这里 |
|------|:---:|------|
| 重试 | 极低 | 类型1是最高频的日常幻觉——先排最便宜的 |
| 拆解 | 低 | 不增加外部信息，只改变输出结构 |
| 分块 | 中 | 需要判断哪些信息可丢弃，或增加分块聚合的推理次数 |
| 锚定 | 低 | 成本极低，但前三种更常见，排最后；同时起排除确认功能 |

关键洞察：**诊断和修复是同步完成的。** 哪一步恢复了正确输出，该步即同时完成了判定和修复。

### 6.3 与当前通行做法的对比

当前用户面对幻觉的典型反应：反复重新生成（假设一切是类型1）；同时加CoT+RAG+分块（成本堆叠，不区分类型）；放弃。诊断树把"碰运气"变成"按顺序二分查找"——每次只加最少干预，用反馈导航。

### 6.4 自动化潜力

该流程可封装为轻量包装层：无需访问模型内部状态，只比对输出，干预模板预置，积累配对后可训练简单分类器跳过试探步骤。对API-only模型适用，跨模型迁移。

诊断树的四步对应Transformer的三个功能层——重试在解码采样层、拆解外挂到FFN知识存储层、分块和锚定在注意力路由层。干预靶点由架构功能决定，而非由经验选择。

---

## 7. 讨论

### 7.1 适用边界

本分类适用于推理时系统性失效，不覆盖训练阶段根因。边界条件是"任务被良好定义"——这意味着分类不声称覆盖所有幻觉，而是覆盖在任务条件明确的前提下仍会出现的幻觉。将训练诱导的对齐偏差和数据投毒排除在外是有意为之：它们的根因在训练目标而非推理架构。

### 7.2 与现有缓解策略的对位

所有常见缓解策略均可在此框架中找到唯一的主靶向类型：RAG同时缓解类型2和类型3，CoT靶向类型2，分块靶向类型3，对比解码靶向类型1，注意力校准靶向类型4。这从侧面验证了分类的操作完整性。

### 7.3 开放问题与后续工作

四分类和诊断树共同指向一个更底层的问题：这四种失效共享同一个结构前提，而非四个独立缺陷。

当前Transformer中，注意力路由寄生在语义系统的参数和表征之上。路由所需的QK匹配信号从token表征向量中读取，该向量同时也服务于FFN的语义变换——两种需求不同质（语义需要高维稠密，路由需要低维结构化），但混在同一个向量里。路由本身没有独立的可训练参数——W_Q和W_K属于语义参数空间，训练时被语义目标的梯度主导更新。解码生成阶段也没有路由层面的反馈监控。路由是三个功能层中唯一"没有自己的东西"的。

四种幻觉是这一结构状况的四个不同出口。现有文献在各出口做了局部缓解（Tang et al., 2025；Guo & Chen, 2025；Gu, 2026；Hsieh et al., 2024），但尚未有工作将"路由的参数独立性和表征独立性"作为正面的架构问题提出。

这引出两个问题——本文不回答，但它们是后续工作的起点：

- **如果给路由一个独立于语义的表征子空间（在残差流中拆分出专属于路由的向量维度，不被语义变换污染），四种失效中有几种会在不依赖任何校准或训练干预的情况下自行消退？**

- **如果路由拥有可训练的策略参数（独立于FFN权重、有自己的学习目标和更新速度），路径缺失能否从一个训练覆盖问题转化为一个参数容量问题？**

如果未来架构解决了路由的独立性问题，诊断树的类型3和类型4可能大幅减少，但类型1和类型2作为更底层的结构和训练问题仍会存在。诊断树的实用价值在该假设下收窄但不消失。

---

## 8. 结论

本文提出了一个LLM系统性幻觉的四分类框架——误差传播、路径缺失、容量溢出、注意力漂移——覆盖了模型推理全链条的失效模式。以主导失效模式为归因标准，四类型近似正交、在给定边界条件下近似完备。基于该分类的最小成本诊断树，以递增成本的简单干预替代复杂的内部指标监测，诊断与修复同步完成。

四个类型对应了Transformer的三个功能层——FFN知识存储、注意力上下文路由、自回归解码——从失效角度展示了它们的功能独立性。更深层地，四种类型共享同一个结构前提：注意力路由在参数和表征上寄生在语义系统之上。这意味着修复不应是四个独立补丁，而是一处结构性修改。后续工作将从此入口展开。

## 附录A：诊断树伪代码

```python
def diagnose_and_fix(model, prompt, verify_fn):
    """最小成本诊断树。
    
    参数:
        model: 任何支持 generate() 的LLM接口
        prompt: 原始用户输入
        verify_fn: 外部验证函数，返回 bool。例如人工确认、规则校验或独立验证模型。
                   验证本身不在本框架范围内。
    """
    MAX_SAFE_TOKENS = 16000  # 保守估计的有效上下文上限，可按模型调整
    
    # Step 1: 重试 — 类型1（误差传播）
    output = model.generate(prompt)
    if verify_fn(output):
        return output, "OK"
    output2 = model.generate(prompt)
    if verify_fn(output2):
        return output2, "Type1: 误差传播，重试修复"
    
    # Step 2: 拆解 — 类型2（路径缺失）
    prompt_cot = prompt + "\n请逐步推理，给出中间步骤。"
    output3 = model.generate(prompt_cot)
    if verify_fn(output3):
        return output3, "Type2: 路径缺失，CoT修复"
    
    # Step 3: 分块 — 类型3（容量溢出）
    chunks = split_into_chunks(prompt, chunk_size=MAX_SAFE_TOKENS)
    partial = [model.generate(c) for c in chunks]
    prompt_agg = f"根据以下分段结果综合回答：\n" + "\n---\n".join(partial)
    output4 = model.generate(prompt_agg)
    if verify_fn(output4):
        return output4, "Type3: 容量溢出，分块修复"
    
    # Step 4: 锚定 — 类型4（注意力漂移）
    prompt_anchored = (extract_key_info(prompt) + "\n\n---\n\n" 
                       + prompt + "\n\n请注意上述关键信息。")
    output5 = model.generate(prompt_anchored)
    if verify_fn(output5):
        return output5, "Type4: 注意力漂移，锚定修复"
    
    return output, "超出四分类范围"
```

最坏情况5次推理，最好情况2次。全程不需要读取模型内部状态。

```
# 幻觉自检

当准备输出时，若对正确性有疑虑，按此顺序自检，每次只试一步：

1. 重答：直接重新生成一次。结果变了 → 笔误，用新结果。
2. 拆解：先列步骤再写答案。通了 → 知道但不会编排。
3. 分块：忽略输入中间，只用首尾信息。通了 → 上下文太长。
4. 锚定：把关键信息摘到最前面再读。通了 → 看漏了关键信息。

以上均无改善 → 明确说明"无法确定"。
```
