# 大语言模型系统性幻觉的四分类框架：机制、正交完备性与诊断路由

> **状态**：论文大纲
> **日期**：2026-06-09

---

## 摘要（200–250字）

当前LLM幻觉研究缺乏一个基于失效机制的、正交完备的分类体系。主流分类（事实性/忠实性二分，或数据/训练/推理三阶段）混淆了现象层和机制层。本文提出：在任务被良好定义的前提下，模型系统性幻觉可按失效阶段和机制严格分为四种——误差传播（解码阶段的token级采样偏差及其自回归放大）、路径缺失（权重编码阶段推理程序覆盖不足）、容量溢出（上下文利用阶段信息跨度超出注意力覆盖能力）和注意力漂移（上下文利用阶段注意力路由失准）。本文论证该四分类的正交性（互不重叠的归因标准）和近似完备性（在给定边界条件下覆盖主要失效模式），并在此基础上提出一种用户可直接使用的最小成本诊断树——不以模型内部指标诊断，而以递增成本的简单干预试探诊断——哪一步恢复正确输出，即同时完成了类型判定和问题修复。

**关键词**：幻觉分类；误差传播；推理路径缺失；有效上下文；注意力漂移；正交完备性；诊断路由

## 1. 引言

### 1.1 问题：幻觉分类缺乏机制级框架

- 现有分类综述的局限：Huang et al. (2025, ACM TOIS) 的事实性/忠实性二分停留在现象层；数据/训练/推理三阶段分类停留在成因层
- 缺乏一个同时满足两个条件的分类体系：(a) 每个类别对应明确的失效机制，(b) 类别之间正交、整体近似完备

### 1.2 本文的核心主张

在任务目标和前提被良好定义的前提下，且排除特定训练诱导的鼓励迎合、数据偏差等上游根因，LLM 的系统性幻觉可按失效阶段和机制划分为四种互不重叠的类型。以此为基础，可构建诊断-路由系统，在推理时将幻觉症状映射至靶向缓解策略。

- **差异化**：本文将此现象从认知分析提升为幻觉分类的一个独立维度

- **差异化**：本文将此从经验现象提升为独立的幻觉类型，区别于容量不足

## 3. 四分类详述

### 3.1 类型1：误差传播（Token-Leakage Propagation）

**定义**：自回归解码中，某步从条件概率分布中偶然采样到一个低概率token（类比笔误），该偏差token进入后续所有步的隐藏状态计算，被自回归机制逐层放大为整段偏离。

**与误差起源的区别**：本类型描述的是**传播/放大机制**，而非偏差为何产生。偏差的起源可能来自类型2-4，但传播机制本身构成一个独立的失效模式——即使起源偏差极小（单token局部偏离），传播后的后果也可能不成比例地被放大。

**数学基础**：
$$P(x_{1:T}) = \prod_{t=1}^{T} P(x_t | x_{1:t-1})$$
若第k步采样到低概率token（ε_k扰动），则第k+1步的条件分布变为：
$$P(x_{k+1} | x_{1:k} + \epsilon_k) \neq P(x_{k+1} | x_{1:k}^*)$$
分布偏移在后续步骤中累积（Arora et al., 2022给出误差上界T²ε）。

**类比**：笔误——不是不会写这个字，只是恰好写错了，但后文可能被这个错误引导偏离主题。

**经验证据**：
- Mekouar (2024)：多模型上验证幻觉token的正自相关（lag-1显著），温度放大该效应
- Zhang et al. (ICML 2024)：GPT-4能识别87%的自身幻觉，但早期幻觉产生后后续错误不可避免

### 3.2 类型2：路径缺失（Missing Reasoning Paths）

**定义**：模型在权重中同时存储语义知识（"是什么"，可通过单步前向传播提取，训练数据覆盖广）和推理路径（"怎么推"，需多步隐含展开，训练数据覆盖密度低）。若某条推理路径在训练数据中覆盖不足或完全缺失，模型在该路径上无正确程序可执行，输出等效于胡言乱语。

**关键区分**：这不是"不知道事实"——模型可能拥有所有相关事实。这是"不知道怎么把这些事实串联成一个有效推理"。

**数学基础**：
- 前馈层（FFN/MLP）存储可跨样本泛化的语义知识——经过预训练，模型参数中编码了词语间的概念关联、事实三元组等信息（Geva et al., 2021；Meng et al., 2022）
- 自注意力机制执行上下文路由——每层根据当前隐藏状态动态计算信息流向，决定哪些token的信息被聚合到当前位置
- 多步推理需要L层中注意力路由的特定序列编排——若某条推理路径对应的路由模式在训练数据中从未出现过，即使FFN中存储了所有需要的子知识，注意力也无法正确编排它们（Brinkmann et al., ACL 2024）

**类比**：学生知道所有概念定义与公式定理，但没学过这类题型的解题路径。

**经验证据**：
- "Reversal Curse" (Berglund et al., 2023)：GPT-4正向79%，反向33%
- "Alice Problem" (Nezhurina et al., 2024)：顶尖模型在组合推理上崩溃至~0%
- Li et al. (LREC-COLING 2024)：程序性知识仅在简单逻辑任务中有效

### 3.3 类型3：容量溢出（Capacity Overflow）

**定义**：有效上下文（注意力实际能高质量聚焦的token范围，通常10K–200K）远小于标称最大上下文（可达1000K）。当任务所需信息跨度超过有效上下文时，关键信息超过注意力覆盖能力，任务无法被可靠完成。

**有效上下文 vs 最大上下文**：
- 最大上下文：架构层面可接受的token上限（由位置编码、KV缓存等硬约束决定）
- 有效上下文：功能层面注意力能**有意义地利用**的token范围（由注意力稀释、位置衰减等软约束决定）
- 经验发现（"Context Is What You Need", 2025）：MECW可低至MCW的1%

**类比**：一本书有1000页，但你真正能有效精读的只有100页，中间900页只是翻过。

**经验证据**：
- Hsieh et al. (ACL 2024)：U形注意力曲线，中间位置信息系统性被低权重
- "Not All Needles Are Found" (2026.01)：更长上下文在信息稀释时可能降低而非提升准确率

### 3.4 类型4：注意力漂移（Attention Drift）

**定义**：即使任务复杂度在有效上下文容量之内，注意力也未必自动落在正确的信息区间。自注意力没有内建的"应该聚焦何处"的保证——它受位置偏差（首因/近因效应）、内容相似度干扰（语义相近但不相关的文本块在注意力竞争中胜出）、以及缺乏显式检索路由的影响，可能聚焦于无关区域而遗漏关键上下文。

**与类型3的关键区别**：
- 类型3：信息量本身超过了注意力容量（"装不下"）
- 类型4：信息在容量内，但注意力没有落在正确的位置（"走神了"）

**类比**：还是那本1000页的书，你看了100页，但是没看目录，走神了没把握到精髓，读完不知道刚才读了什么。

**经验证据**：
- Liu et al. (2023)："Lost in the Middle"效应——关键信息在中间位置时检索准确率显著低于两端
- Chuang et al. (2024)：注意力回看率（lookback ratio）下降是上下文幻觉的实时预测信号
- An et al. (NeurIPS 2024)：IN²训练通过信息密集型长上下文QA修复注意力覆盖，效果显著

### 3.5 四种类型总览

| # | 类型 | 失效阶段 | 失效机制 | 类比 | 典型触发条件 |
|---|------|---------|---------|------|-------------|
| 1 | 误差传播 | 解码生成 | token级采样偏差的自回归级联放大 | 笔误 | 高熵状态、高温度采样 |
| 2 | 路径缺失 | 权重编码 | 推理程序的训练覆盖不足 | 没学过这知识 | 长尾/组合推理任务 |
| 3 | 容量溢出 | 上下文利用 | 信息跨度 > 注意力覆盖能力 | 注意力不够 | 长文档推理、多文档综合 |
| 4 | 注意力漂移 | 上下文利用 | 注意力路由失准 | 走神 | 语义干扰、位置偏差 |

## 4. 正交性论证

### 4.1 正交性定义

给定一个幻觉实例，存在一套明确的诊断标准使其主要归入且仅归入一个类型。

### 4.2 诊断归因标准

**判归类型1（误差传播）**：模型具备完成任务所需的知识和上下文——即有正确的推理路径可用（非类型2），关键信息在上下文容量内且已被注意力正确读取（非类型3、4），但在某解码步偶然采样到低概率token，后续输出被放大偏离。

- 诊断信号：logits在各步的熵值分布——某一步熵突然升高（选择不确定性大），后续步逐步偏离
- 可区分性：若在偏离发生前截断并重新采样，能以较高概率回到正确轨道

**判归类型2（路径缺失）**：所有相关信息已在上下文中，注意力也正确覆盖了这些信息（非类型3、4），但模型在从信息到结论的推理步骤中存在系统性错误——且该错误在多次独立采样中一致出现（非类型1的偶然性）。

- 诊断信号：多次独立采样输出的一致性错误模式（排除了单次采样偶然性）
- 可区分性：提供额外的中间推理步骤（chain-of-thought提示）可在某些情况下修复——说明子能力存在但更高级路由未 正确编排

**判归类型3（容量溢出）**：任务信息跨度经量化超过模型的已知有效上下文范围，且使用上下文压缩/分块策略后性能显著提升。

- 诊断信号：信息分布在token序列中的跨度与模型已知的有效上下文范围的关系
- 可区分性：将内容分块独立处理后聚合结果是否显著改善——若是→容量溢出

**判归类型4（注意力漂移）**：信息在有效上下文范围内（非类型3），模型也具备相关推理能力（非类型2），但在多个独立采样中一致性地忽略某些关键上下文区域（非类型1的偶然性）。

- 诊断信号：注意力权重在关键信息区间的分布异常——低覆盖或高覆盖但位置错误
- 可区分性：显式标记关键信息（如高亮/重复提示）后是否显著改善——若是→注意力漂移

### 4.3 正交性结论

四类型有不同的归因依据和独立的诊断信号。类型1和类型2-4不处于同一因果层级（1是传播机制，2-4是起源机制），但这不影响归因的唯一性——一个案例可以被明确判定为"路径缺失导致错误 + 误差传播机制放大了后果"，其中"路径缺失"是主导类型，"误差传播"是次要传播因素。在正交性论证中，我们以**主导失效模式**为归因标准。

## 5. 完备性论证

### 5.1 边界条件

本分类体系的完备性受以下边界条件约束：
- "任务被良好定义"：排除对齐诱导幻觉（RLHF迎合、不确定性压制）
- 排除训练数据投毒、错误标注等上游根因（这些是四种类型的诱发条件，而非失效机制本身）
- 聚焦于推理时系统性失效，不覆盖对抗攻击、分布外漂移等非标准输入场景

### 5.2 覆盖范围

四种类型覆盖了模型推理的**全部三个关键阶段**：
- 权重编码阶段（类型2）
- 上下文利用阶段（类型3+4）
- 解码生成阶段（类型1）

### 5.3 可能遗漏的类型及论证

**Q1：知识冲突（权重先验与上下文事实冲突）→ 是否构成独立类型？**
- 论证：知识冲突的失效表现在推理时有两种情况——(a) 模型错误地选择了权重先验而忽略上下文→归入类型4（注意力漂移到权重先验）；(b) 上下文权威性在任务定义中已明确→不属于"任务良好定义"范畴
- 结论：在本文边界条件下，不构成独立的第五类型

**Q2：组合组装失败（子路径存在但组合为新推理链时出错）→ 是否构成独立类型？**
- 论证：子路径存在但组合失败，本质上仍然是推理路径的不完整——注意力路由无法正确编排各子路径的顺序和条件。归入类型2的扩展含义
- 结论：归入类型2，不构成独立的第五类型

**Q3：校准性解码错误（非小概率token，而是logits本身指向错误方向）→ 是否构成独立类型？**
- 论证：若logits指向错误方向且非偶然性（多次采样一致）→根源在权重编码阶段→归入类型2。若logits指向正确但单次采样偏差→类型1。类型1的定义核心是"传播/放大"，不要求错误token必须是低概率——只要求它是局部偏差
- 结论：归入类型1（若采样偏差是主导因素）或类型2（若权重编码是主导因素），不构成独立的第五类型

### 5.4 完备性结论

在给定边界条件下，四分类覆盖了模型推理全链条上的系统性失效模式。可能存在边界子情况（如类型1和类型2在极端低概率token+路径缺失同时出现时的归因模糊性），但这些属于归因精度问题而非分类覆盖问题。

## 6. 诊断路由：最小成本诊断树

上述四分类的直接应用不是构建复杂的内部状态监测系统，而是一个任何用户都能立即使用的**诊断树**。核心原则只有一条：

> **不以内部指标诊断，以干预反馈诊断。哪个手段有效，就暴露了真实的幻觉类型。**

### 6.1 诊断树：四步递增成本试探

发现模型输出有问题后，按以下顺序依次尝试。成本从低到高排列——每一步的成本都比"从一开始就全上"的做法低得多，也避免了"用RAG修笔误"式的错配浪费。

```
发现幻觉
  │
  ├─ 第一步：重新生成（成本：1次推理）
  │   同一输入再问一次。
  │   ├─ 恢复正常 → 类型1（误差传播/笔误）。完成。
  │   └─ 仍然错误 → 不是偶然采样偏差，继续。
  │
  ├─ 第二步：拆解步骤（成本：添加"请逐步推理"）
  │   要求模型给出中间步骤。不加任何新信息。
  │   ├─ 恢复正常 → 类型2（路径缺失）。模型有子能力但不会编排。
  │   │               CoT已将长程推理路径拆为模型已知的短程子步骤。
  │   └─ 仍然错误 → 不是路径编排问题，继续。
  │
  ├─ 第三步：缩短输入（成本：截断上下文 + 1次推理）
  │   丢弃中间段落，只保留首尾关键信息。或分块独立处理后聚合。
  │   ├─ 恢复正常 → 类型3（容量溢出）。信息在完整时太多，
  │   │               截断/分块后每段在有效上下文范围内。
  │   └─ 仍然错误 → 不是容量问题，继续。
  │
  └─ 第四步：锚定关键信息（成本：修改提示词 + 1次推理）
      将关键信息移到开头或结尾，或显式标记（"请注意以下内容：……"）。
      ├─ 恢复正常 → 类型4（注意力漂移）。信息一直在，但模型一直在看别处。
      └─ 仍然错误 → 超出四分类范围
                     → 可能原因：权重中编码了错误知识、任务本身无解、
                        或为上游根因（训练数据错误/对齐偏差）
```

### 6.2 为什么这个顺序

| 步骤 | 成本 | 逻辑 |
|------|:---:|------|
| 重新生成 | 极低 | 类型1（笔误）是最高频的日常幻觉类型——先排最便宜的 |
| 拆解步骤 | 低 | 把长程推理拆为短程子步骤，不增加任何外部知识和上下文成本 |
| 缩短输入 | 中 | 需要判断哪些段落可丢弃，但推理本身不增加token |
| 锚定信息 | 极低/单次 | 只改提示词，不增加推理量；但排在第三步之后——因为前两种更常见 |

关键洞察：**诊断和修复是同步完成的。** 哪一步恢复了正确输出，该步即同时完成了"判断类型"和"解决问题"两件事。不需要一个独立的"先诊断、再开方"阶段。

### 6.3 诊断树 vs. 当前通行做法

当前用户面对幻觉的典型反应：
- 反复重新生成（假设一切是类型1）
- 同时加CoT+RAG+分块（不区分类型，成本堆叠）
- 放弃，认为"模型不行"

诊断树把"碰运气"变成了"按顺序排查"——每次只加最少的干预，用反馈来导航。本质上是一个**输入空间中的二分查找**：先区分采样/非采样，再区分编排/容量，最后区分容量/注意力。

### 6.4 自动化潜力

上述流程目前以手动方式实现，但其核心逻辑可被封装为一个轻量包装层：

- **无需访问模型内部状态**：不读logits、不读注意力权重、不标定MECW
- **只比对输出**：原始输出 vs. 带干预的输出，正确与否由用户确认或校验函数判定
- **干预模板预置**：四种干预对应四个标准化的提示词变换
- **学习型路由**：积累足够的诊断-干预配对后，可通过简单分类器跳过试探步骤，直接从输入特征预测干预类型

这一路线的关键优势：对API-only模型适用，干预模板跨模型迁移，不依赖任何模型内部信息。

### 6.5 与Transformer架构分层的关系

诊断树的四步恰好对应了Transformer的三个功能层，每一步干预的靶点在架构中有明确对应：

| 步骤 | 干预靶点 | 架构层 |
|------|---------|--------|
| 重新生成 | 改变采样路径 | 解码采样层 |
| 拆解步骤 | 外挂推理路径编排 | FFN知识存储层（路径不在权重中，用文本补） |
| 缩短输入 | 缩小路由搜索空间 | 注意力上下文路由层 |
| 锚定信息 | 纠正路由偏差 | 注意力上下文路由层 |

干预靶点由架构功能决定，而非由经验选择——这正是四分类能够系统化（而非随机）地指导干预的原因。

## 7. 讨论与边界

### 7.1 框架的适用边界

- 本分类体系适用于**推理时系统性失效**，不覆盖训练阶段根因
- 边界条件（任务良好定义）的进一步精确化
- 四类型在实际部署中的频率分布估计

### 7.2 诊断树的局限与扩展方向

- 第四步失败后的"超出分类范围"路径需要更细化的处理——目前退化为人工判断
- 多类型并发（如类型2+类型4同时存在）时单步修复不足以完全恢复，需要组合干预
- 诊断树的深度（4步）可随分类细化而扩展，但成本递增原则不变
- 可与RL-based路由器集成——将诊断树的选择逻辑训练为策略（参见RAL, Li et al. 2026），跳过试探步骤直接从输入特征预测最优干预

### 7.3 与现有缓解策略的映射

现有常见缓解策略在四类型框架中的位置：
- RAG → 同时缓解类型2（外挂知识）和类型3（缩小上下文）
- CoT → 靶向类型2（显式化推理路径）
- 分块处理 → 靶向类型3
- 对比解码 → 靶向类型1
- 注意力校准 → 靶向类型4

当前所有缓解策略均可在此框架中找到唯一的主靶向类型——这从侧面验证了分类的完备性和操作性。

### 7.4 开放问题

四分类和诊断树共同指向一个更底层的问题——这四种失效共享同一个结构前提，而非四个独立缺陷。

当前Transformer中，注意力路由寄生在语义参数和语义表征之上：路由所需的QK匹配信号从同一个token表征向量中读取，该向量同时也服务于FFN的语义变换；路由本身没有独立的可训练参数——W_Q和W_K属于语义参数空间，训练时被语义目标的梯度主导更新；解码生成阶段没有任何路由层面的反馈监控。路由是三个功能层中唯一"没有自己的东西"的——没有自己的参数、没有自己的表征空间、没有自己的反馈回路。

四种幻觉在这一视角下是同一结构状况的四个不同出口。现有文献在每个出口各自做了局部缓解（Tang et al., ICML 2025；Guo & Chen, 2025；Gu et al., 2025；Hsieh et al., ACL 2024），但尚未有工作将"路由的参数独立性和表征独立性"作为一个正面的架构问题提出。

这引出两个承接性问题——本文不回答，但它们是后续工作的起点：

- **如果给路由一个独立于语义的表征子空间（在残差流中拆分出专属于路由的向量维度，不被语义变换污染），上述四种失效中有几种会在不依赖任何校准或训练干预的情况下自行消退？**

- **如果路由拥有可训练的策略参数（独立于FFN权重、有自己的学习目标和更新速度），推理路径缺失能否从一个训练覆盖问题转化为一个参数容量问题——即从"数据中没见过这条路径"变为"参数还没学会这条路"？**


---

## 8. 结论

本文提出并论证了一个LLM系统性幻觉的四分类框架——误差传播、路径缺失、容量溢出、注意力漂移——覆盖了模型推理全链条的失效模式。论证了该分类在"任务良好定义"边界条件下的正交性和近似完备性。基于该分类，提出了最小成本诊断树——以递增成本的简单干预（重试→拆解→缩短→锚定）替代复杂的内部指标监测，诊断与修复同步完成。该框架为幻觉缓解从"碰运气式堆叠策略"转向"按顺序最小干预"提供了理论基础。

四个类型分别对应了Transformer的三个功能层——FFN知识存储、注意力上下文路由、自回归解码——从失效的角度展示了这三层的结构独立性。更深层地，四种类型共享同一个结构前提——注意力路由在参数和表征上寄生在语义系统之上，没有独立性。这指向一处结构性修改而非四个独立补丁。后续工作将从这一入口展开。

## 附录A：诊断树伪代码

```python
def diagnose_and_fix(model, prompt, verify_fn):
    """最小成本诊断树。verify_fn 可以是用户确认或自动校验函数。"""
    
    # Step 1: 重试 — 诊断类型1（误差传播）
    output = model.generate(prompt)
    if verify_fn(output):
        return output, "OK"
    output2 = model.generate(prompt)  # 不做任何修改，直接重新生成
    if verify_fn(output2):
        return output2, "Type1: 误差传播（笔误），重试修复"
    
    # Step 2: 拆解 — 诊断类型2（路径缺失）
    prompt_cot = prompt + "\n请逐步推理，给出中间步骤。"
    output3 = model.generate(prompt_cot)
    if verify_fn(output3):
        return output3, "Type2: 路径缺失，CoT修复"
    
    # Step 3: 缩短 — 诊断类型3（容量溢出）
    chunks = split_into_chunks(prompt, chunk_size=MAX_SAFE_TOKENS)
    partial_results = [model.generate(c) for c in chunks]
    prompt_aggregate = f"根据以下分段结果综合回答：\n" + "\n---\n".join(partial_results)
    output4 = model.generate(prompt_aggregate)
    if verify_fn(output4):
        return output4, "Type3: 容量溢出，分块修复"
    
    # Step 4: 锚定 — 诊断类型4（注意力漂移）
    # 将关键约束/信息移到prompt最开头
    prompt_anchored = extract_key_info(prompt) + "\n\n---\n\n" + prompt + "\n\n请注意上述关键信息。"
    output5 = model.generate(prompt_anchored)
    if verify_fn(output5):
        return output5, "Type4: 注意力漂移，锚定修复"
    
    # 超出分类范围
    return output, "超出四分类范围（可能为权重知识错误或上游根因）"
```

成本分析：最坏情况5次推理（1次原始+4次试探），最好情况2次（重复生成成功）。所有步骤均不需要读取模型内部状态。

## 附录C：幻觉自检 Skill

将以下提示词放入 Agent 系统提示中即可生效：

```
# 幻觉自检

当你准备输出时，如果对正确性有疑虑，按此顺序自检，每次只试一步：

1. 重答：直接重新生成一次。如果结果变了 → 笔误，用新结果。
2. 拆解：先列出解题步骤，再按步骤写答案。如果通了 → 会用。
3. 缩上下：忽略输入中间部分，只用首尾信息。如果通了 → 太长。
4. 锚关键：把关键信息摘出来放在最前面，再读一遍。如果通了 → 看漏。

四步都不通 → 诚实告诉用户"这个我不确定"。
```
