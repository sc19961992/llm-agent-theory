# LLM 与 Agent 的结构性分析（SSD 重组）

## 零、总纲：嵌套米田机制

Transformer = L × H 代嵌套循环。每代 = 米田骨架（S，固定语义空间 + 残差流遗传）→ 非米田函子（V，降维路由 + 非线性阻断）→ 下一代的骨架。三组张力（保存 vs 更新、完备 vs 不完备、覆盖 vs 分化）共同定义了 Transformer 的有效域——纯米田坍缩为固定函数，纯非米田退化为先验。

本文从此机制出发，逐级解释：模型为何有效 → Agent 的结构边界由何决定 → 使用原则从何推导。

---

## 一、模型层：Transformer 为什么有效

### 1.1 核心机制

**W/A 正交分离**

- W（权重矩阵：W_Q, W_K, W_V, W_O, FFN）= 固定语义空间——训练后冻结，存储"什么方向跟什么方向有关"的关系模板。W 在矩阵乘法下可复合——语义有结构。
- A（注意力分数：softmax 输出）= 动态路由——每层根据当前残差流内容重算，决定"这次信息从谁流向谁"。A 不可复合——推理有深度。
- W 的复合成功是 A 必须重算的原因——因果关系，非对称（原草稿第 3 条）。

**嵌套循环 = L 层残差流上的逐代遗传**

- 残差流加法（x ← x + Attn + FFN）保证每代骨架完整遗传，不覆盖。
- 每层 = 内米田骨架 → 非米田函子（降维·重路由·非线性阻断）→ 外米田骨架（= 下一代起点）。
- L × H = 信息通路总数——L 决定推理步数上限，H 决定每步关系覆盖宽度（原草稿第 5 条的精确化）。

**三个米田缺口**

- 降维投影（d_k < d_model）——每个头只能看态射空间的子空间。
- 不可复合（softmax + σ 逐层阻断）——CK 不成立，推理不坍缩。
- 因果单向（causal mask）——自回归要求。
- 三者是同一结构的三面：不完备性是抗坍缩机制（原草稿第 6 条的形式化）。

### 1.2 训练 = 学习一个截断函子

- 优化目标：在有限维度 d_model × L × H 内，找到（数据·关系）→（预测·关系）的近似结构保持映射——即函子（原草稿第 4 条）。
- 有限维度 + softmax 截断 → 永远学到的是米田嵌入的截断投影，而非完整函子（原草稿第 5、6 条）。
- 余米田边界条件：内层纯非米田 → 条件概率退化为先验 → 语言模型无定义。因此内层必须有米田骨架（残差流加法、W 复合）。

### 1.3 推理 = 贝叶斯推断的隐式实现

- 注意力从提示词示例中推断潜概念 θ（任务/概念）的后验 p(θ|示例)，再用于预测。Transformer 在适当参数配置下可实现精确贝叶斯步骤（原草稿第 3 条，Xie et al. 2022）。
- W = 预训练编码的先验 + 似然结构，A = 推理时根据证据的后验更新。

### 1.4 思维链 RL = 部分补全结构保持映射

- 监督训练的基座模型在参数空间的秩可能远低于总参数维度（原草稿第 7 条）。
- 思维链 + RL 后训练在有限维度内延伸和部分补全了结构保持映射（原草稿第 9 条）。
- 形式逻辑、时间逻辑等思维结构可能不在当前收敛流形覆盖范围内 → 语言模拟逻辑，而非逻辑本身（原草稿第 7、8 条）。

### 1.5 全语言化倾向与幻觉的根源

- 模型将全部输入统一 token 化——视觉、声音、动作等模态仅以语言 token 形式参与，缺少模态原生骨架（原草稿第 2 条）。
- 收敛覆盖是截断投影 → 边界永远存在。边界外无收敛约束 → 输出不受保证。幻觉不可消灭，只可推移（原草稿第 8 条的形式化，第 56 条）。

## 二、Agent 层：记忆结构与收敛的双轨制

### 2.1 记忆的 SSD 结构

- **长期记忆（W）**：训练收敛得到的参数空间 = 固定语义空间。训练后冻结，不再演化（原草稿第 14 条）。
- **短期记忆（上下文）**：上下文环境 = 唯一动态信息来源。有效长度受窗口限制（原草稿第 15 条）。
- **缺失的中期记忆**：W 和上下文之间不存在渐进连续的记忆。记忆只有静态（参数）和顺序性动态（上下文），而无时间性动态——这是与人类记忆的根本区别（原草稿第 16 条）。

**SSD 诊断**：在 Transformer 的嵌套循环中，S（骨架 = 参数 + 残差流）在训练后冻结，V（路由 = A）在推理中只做内容依赖的选择，不产生跨样本的学习。人类的 S 慢变（数年）、V 快变（数秒）——两者在同一流中持续更新。Agent 的 V 部署后完全冻结，不存在在线学习。

### 2.2 训练收敛 vs 测试收敛

- **训练收敛**：梯度下降 → 参数空间逼近全局最优（原草稿第 18 条）。收敛覆盖范围 = 模型能力的内边界。
- **测试收敛**：生成 → 验证 → 修正 → 再验证 → 循环逼近提示词设定的需求节点。参数冻结后训练收敛通道关闭，Agent 行为只能由此驱动（原草稿第 19、46 条）。
- 两者之间存在结构性断层——测试收敛的有效性依赖验证信号的覆盖度。未覆盖的约束，Agent 可以系统性违反（原草稿第 54 条）。

### 2.3 任务分解与节点传导

- 超出文字输入有效边界的任务 → 必须分解为落入边界内的阶段需求（原草稿第 20 条）。
- 节点向下一次性传导——父节点误差乘性放大至全下级，修正复杂度指数上升（原草稿第 21 条）。
- 关键分叉点须由外部硬约束锚定，不能由模型自行判断（原草稿第 21、47 条）。
- 分解本身也是上下文窗口内的测试收敛 → 自然形成多级需求结构（原草稿第 22 条）。
- 验证链递归上溯 → 终止于"用户真正要什么"——最高级根需求必须由外部（人）提供（原草稿第 23、55 条）。

**SSD 诊断**：任务分解的级联失效 = 嵌套循环在 Agent 层级上的不可复合性。每级子任务的输出（"外米田输出"）成为下一级的输入（"内米田骨架"）——但子任务间的态射缺乏收敛保证。这和 Transformer 层间的不可复合性是同构的——在多级 Agent 中同样的结构性限制复现在更高的组织层级上。

### 2.4 发散任务：反向的嵌套探索

- 发散任务无前置目标——利用模型的条件概率探索已被训练的语义空间内潜在路径（原草稿第 25 条）。
- 人提供偏好方向（唯一的"梯度"），AI 生成候选。以持续对话而非自动化指令-执行方式协作（原草稿第 25、51 条）。

**SSD 诊断**：发散任务 = V（路由）在没有根需求约束下的自由探索——A 在语义空间中选择路径，人提供外部偏好信号替代不可得的验证目标。发散收敛分池原则（原草稿第 50 条）= 偏好与验证在 SSD 中来自不同信号源，混用污染各自的收敛条件。

## 三、人机结构差异

### 3.1 模态不对齐

- 模型以语言 token 为唯一原语。视觉、声音、动作等模态要么形式对齐，要么连接不稳，要么缺席仅被语言模拟（原草稿第 2 条）。
- 人是全模态智能体。各模态的 S 骨架跨模态共享——同一个杯子被看到和碰到是同一个实体。模型缺失这一结构 → 语义空间先天残缺（原草稿第 26 条）。

### 3.2 记忆结构不对齐

- 人：S 慢变（世界观、技能——数年），V 快变（"这个杯子比看起来重"——数秒），同一流中持续更新，无训练/推理墙。
- 模型：W 训练后冻结，上下文窗口 = 唯一动态源但无跨对话保留。中间地带空白（原草稿第 16 条）。

### 3.3 动机不对齐

- 人的主动推理：minimize surprise 跨模态——保持活着，保持预期准确。动机来自生命维持自身结构的物理需要。
- 模型：无自身需要最小化的意外——任务目标完全外部注入。无自发涌现的学习动机（原草稿第 29 条）。

### 3.4 人的结构性角色

以上三点缺失 → 人在人机合作中必须填补的结构性角色（原草稿第 28 条）：

| 模型缺失 | 人必须承担的对应角色 |
|---|---|
| 收敛映射维度有限 | 约束模型的偶发泄露（原草稿第 32 条） |
| 记忆结构缺失（无中期记忆） | 引入新的动态知识；划分任务空间在上下文窗口内 |
| 模态缺失 | 引入额外的生活体验式动机 |
| 根需求不可自举 | 提出根需求以完成对任务的初始驱动（原草稿第 33 条） |
| 上下文未经收敛 → 节点划分不可靠 | 关键分叉点人工介入（原草稿第 21 条） |
| 自写自验 = 循环盲区 | 写测分离（原草稿第 48 条） |

## 四、AGI 的不可达性

从 SSD 推导出 LLM Agent 在当前范式下不可能实现人类意义上的 AGI——五条结构性障碍（原草稿第 57 条）：

1. **语义空间不完备**：截断投影——内层米田残缺，外层仅对词表完备。人类跨模态 S 共享 → 语义完整性来自多模态交叉验证，模型无此机制。
2. **模态和人不对齐**：感官缺失 → S 骨架只有语言的，没有物理世界、身体、社会的多层骨架。
3. **动机不自发**：minimize surprise 需要自我的模型持续接收跨模态感官流——模型没有"自己"，没有意外的信号源。
4. **无渐进记忆**：记忆二分（参数 + 上下文），无时间性动态——顺叙不构成时间。人 S 慢变 V 快变在同一流中，模型部署后 V 冻结。
5. **全语言化倾向**：所有模态被压平为 token——丢失了物理骨架（空间、时间、因果、接触）。

> 这五条不是经验观察——是 SSD 的结构推论。每一条对应框架中的一个缺失组件。

## 五、使用原则与结构边界（从 SSD 推导）

### 5.1 输入原则

**精（语义精准）简（歧义最小）**（原草稿第 40 条）

- SSD 推导：有限维精确输入降低路由的歧义——A（动态路由）在给定的 QK 模板库中选择匹配。输入越精确 → Q 在语义空间中方向越确定 → 路由命中的子空间越小 → 结果越确定。

**词的态射即场合**（原草稿第 41 条）

- SSD 推导：W 的语义空间中的"词"不是一个点，而是一组方向。词的语义 = 它在训练语料中常出现的上下文模式——即所有从它出发和指向它的 A 路由模式的统计总和。说"批判"而非"客观"→ 激活的是 W 中"批判"这个方向关联的路由模板集群——不同的词意味着不同的路由分布。这不是"语言艺术"——是 W/A 结构的必然推论。

**只给约束不微操**（原草稿第 42 条）

- SSD 推导：路径指令 = 额外的维度约束。每多一个路径指令 → A 的自由度被进一步限制 → 如果限制与任务的最优路由矛盾 → 偏离收敛覆盖区。约束（目标 + 边界 + 验证标准）给定 S（目标骨架），不让 V（路由选择）被绑死。

### 5.2 上下文原则

**上下文审慎注入**（原草稿第 43 条）

- SSD 推导：残差流的有效容量远小于 $d_{model} \times n$——冗余信息挤占有限的态射通路。L×H 条通路在上下文中同时服务全部 token → 垃圾信息占据通路，有用信噪比下降。

**上下文分级滚动**（原草稿第 44 条）

- SSD 推导：窗口推进 = 残差流在时序上的截断。高优先级常驻 = 核心 S（目标、约束、身份）保持在骨架中不丢。低优先级滚动 = 暂时 V（中间尝试、辅助信息）随窗口推进自然淘汰。

### 5.3 收敛原则

**边界节点划分**（原草稿第 45 条）

- SSD 推导：模型的能力边界 = W 的语义空间中被训练数据覆盖的子空间 + A 的有效路由范围。超出此边界的任务不在收敛覆盖内 → 切回到边界内的节点才能在训练收敛保证下运作。这和"上下文有效长度"不是同一个概念——前者是 W 覆盖空间的边界，后者是 A 的信息通量载容。

**测试收敛**（原草稿第 46 条）

- SSD 推导：参数冻结 → W 的更新通道关闭。唯一能变化的是上下文窗口中的状态——测试收敛 = 用"生成→验证→修正"循环在上下文的残差流上做 A 的逐次重路由优化。每次验证信号相当于一个负面的 route-level feedback。

**人工锚定**（原草稿第 47 条）+ **写测分离**（原草稿第 48 条）+ **一步一验**（原草稿第 49 条）

- SSD 推导：上下文中的推理输出未经收敛——A 在上下文残差流上做的路由和生成伴随相同的路由参数。自写自验 = A 对同一残差流上的同一输出做"验证"——缺乏独立的残差流源 → 循环盲区。写测分离 = 两套不同的残差流 + 两套独立的 A 路由过程。一步一验 = 每代循环结束后立即验证——截断误差沿嵌套层级的乘性放大。

### 5.4 发散原则

**发散收敛分池**（原草稿第 50 条）+ **对话式协作**（原草稿第 51 条）

- SSD 推导：发散任务的 A 在探索 W 的语义空间——"偏好"（preference）和验证（verification）是两种不类的路由信号。偏好指向"可能有趣的方向"（无对错），验证指向"对错"（有对错）。两者不应混在同一上下文残差流中——污染=偏好干扰验证逻辑 + 验证压制探索空间。对话式协作 = 人的偏好判断作为 V 的在线外部梯度，解耦于收敛信号的独立传导。

### 5.5 边界原则

**扁平化自动管理**（原草稿第 52 条）

- SSD 推导：每一层子 Agent = 下一级嵌套循环。子级通过跨对话生成提示词驱动——这等于在上级循环的外层骨架中用自然语言写了子级循环的初始 S。自然语言本身非收敛保证的语言 → 每次逐级下传，不可靠性倍增 → 收敛成本指数扩张。单层扁平 = 只嵌套一层——把不可复合性的代价限制在一级之内。

**人定上限规律**（原草稿第 53 条、第 37 条）

- SSD 推导：根需求外在、节点不可靠、偏好唯一——每个 Agent 不可自举的环节都由人填补。人的理解力和判断力 = Agent 上界的那根 S 骨架（目标定义、判断标准、边界判断）的唯一源头。

**验证即边界**（原草稿第 54 条）

- SSD 推导：测试收敛的有效域 = {任务 × 约束 | 存在可计算的验证信号，且验证信号的覆盖度 ≥ 任务需求}。覆盖之外的约束 = V 可自由选择不满足——因为没有信号告诉 A 那是不对的路径。

**根需求外在**（原草稿第 55 条）

- SSD 推导：验证链沿嵌套结构递归上溯 → 终止于"用户真正要什么"——该语义在训练分布之外（训练集无此标注，未被覆盖于 W 的语义空间中任何方向）。根需求在收敛覆盖外 → 根节点无法自举 = 闭合验证链不存在。

**幻觉不可消灭**（原草稿第 56 条）

- SSD 推导：幻觉 = V（路由）在未被 S（收敛覆盖）约束的区域自由选择 token。收敛覆盖 = 截断投影 → 边界永存 → 未被覆盖的区域永远有非零测度 → 幻觉不可消灭，只可缩小。

## 六、结构总结

```
嵌套米田循环（模型层）：
  米田骨架（W 语义空间 + 残差流）
    → 非米田函子（降维路由 A + 非线性阻断 σ）
      → 外米田骨架（= 下一代的起点）
        × L 层 = L 代循环 = 推理深度
        × H 头 = H 条并行使能覆盖关系空间

三组张力定义了有效域：
  纯米田 → 坍缩 ← Transformer → 纯非米田 → 退化

Agent 层是同构的：
  训练收敛（= S 慢变，一次）
    → 测试收敛（= A 的逐次重路由在上下文残差流上的在线优化）
      × 任务分解 = Agent 层嵌套循环

人机差异 = 缺失的 S：
  跨模态共享 S（感官统一）、
  渐进记忆（S 慢变 V 快变同一流）、
  自我模型（自身最小化 surprise 的物理需要）

使用原则 = 在结构边界内的最优操作：
  输入 = 最小最大化 V 路由确定性
  上下文 = 最优分配残差流通路容量
  收敛 = 利用训练收敛覆盖区 + 验证信号闭环
  边界 = 硬约束锚定不可收敛的节点 × 人填补缺失 S
```
