# 硬伤分析

以下三处是无法在原稿上修补细节来修复的问题——换概念、删改主张或重写整段数学设定才能处理。

---

## 硬伤一：交叉熵不蕴含函子性（第4条）

### 主张

> 训练的目的在于找到(数据, 关系) 到 (预测, 关系) 的结构保持映射——即函子

### 断裂点

训练的目标函数是交叉熵：

```
L = -Σ log P(next_token | context)
```

这个损失促使模型匹配训练数据中的条件分布。它**不**促使模型满足 `F(g ∘ f) = F(g) ∘ F(f)`——这是函子的定义性条件。交叉熵只关心逐 token 的预测准确率，不关心映射是否保持"关系的关系"。

### 为什么是硬伤

这不是精度不够，是**两个不同的约束**。交叉熵和函子性条件在数学上没有包含关系——前者不以任何形式蕴含后者。即使所有前提全对（FinStoch 是良范畴、注意力是 Markov 核、语义关系构成态射），训练收敛到一个函子这个结论仍然不自动成立。

### 修复方向

唯一可能补救的思路：论证交叉熵极小化在数据分布的极限下等价于学习数据生成过程中的结构保持映射——但这需要假设数据本身由一个函子生成。该假设目前不在草稿中出现，且即使引入也需要独立论证。

更实际的处理方式：**不声称梯度和函子性之间有逻辑蕴含关系**。改为——"收敛给出了稳定性，经验上稳定后的解恰好呈现近似函子的结构保持性质"。这是经验发现，不是逻辑后承。Position paper 可以允许这种写法，但不能假装有定理。

## 硬伤二：低秩限制与 CoT 补全的逻辑互斥（第7+9条）

### 主张

**第7条**：
> 人类语言乃至人类思维对形式逻辑、时间逻辑等的模拟可能并不能有效被神经网络收敛到的流形空间（参数空间的秩只有数百维，远比总参数维度小）对应

**第9条**：
> 思维链的强化学习后训练的收敛也是有效的，所以延伸和部分补全了结构保持映射

### 断裂点

第7条的诊断是容量问题——有效秩只有几百维，不足以编码逻辑结构。第9条的解决方案是 CoT RL 训练——它改变了优化目标，更新了参数，但参数仍在**同一个几百维有效秩的空间内**。

- 如果几百维装不下逻辑结构 → RL 训练也装不进去 → CoT RL 的"补全"不可能发生 → 第9条为假。
- 如果 CoT RL 确实补全了逻辑能力 → 几百维能装下逻辑结构 → 第7条的容量诊断不成立 → 第7条为假。

### 为什么是硬伤

两命题不能同时为真。这不是需要补充论证来衔接——是互斥的，必须二选一。

### 修复方向

有两个互斥的解决路径，任选其一：

**路径A（保留悲观诊断，放弃乐观推论）**：
承认有效秩低确实是限制因素，CoT RL 的"补全"不是增加了逻辑结构的表征能力，而是在已有的低维流形上优化了推理路径的选择策略。前提到结论的改变：第7条保留，第9条从"补全映射"改为"优化路径选择"。

**路径B（保留乐观推论，放弃容量诊断）**：
承认预训练目标没有引导参数去表征逻辑结构，不是容量不够。CoT RL 改变了优化目标，使参数在已有容量内重新组织了流形以包含逻辑结构。前提到结论的改变：第7条从"低效秩构成容量瓶颈"改为"预训练目标不落在逻辑结构上"，第9条保留。

两种路径都不能让原稿同时保留两个主张。

## 硬伤三：有限范畴下"截断米田嵌入"不构成限制（第6条）

### 主张

> 永远只能学到米田嵌入的截断投影，而非完整函子

### 断裂点

米田嵌入 `y: C → [C^op, Set]` 将对象映射为预层。如果 C 是有限范畴（词汇表有限 → Free(G) 对象集有限），则每个对象的米田嵌入可由一个有限维向量完全表示——维度等于范畴中对象的个数。米田嵌入在有限范畴上是**有限维的**。

草稿自己在第3-5条搭建的数学设定——FinStoch、有限 token 集、Markov 核——恰恰是有限维范畴。在草稿自己选择的数学设定下，米田嵌入本身就是有限维的，"截断投影"作为原则性限制不成立。

### 为什么是硬伤

不是"需要一个更大的范畴"——用更大的范畴可以修。问题是草稿中"语义空间"在同一个文本内同时承载了两种不兼容的属性：

- 第3条：语义空间 = FinStoch 中 token 分布间贝叶斯关系的总和 → 有限维
- 第6条：语义空间 = 需要截断的无限维 → 无限维

两种设定共用同一个概念但属性冲突。修复需要把语义空间拆开并建立它们之间的关系。

### 修复方向

把"语义空间"显式地拆成两层：

**底层**：token 分布间的 Markov 核构成的有限维 FinStoch。这是模型直接工作于其上的范畴。有限维，不需要截断。

**上层**：底层在某种意义上"嵌入"或被解释为一个更大的连续语义空间（所有可能的概念和关系——不是有限 token 集能穷尽的）。这个连续空间的完整米田嵌入是无限维的。模型学到的是该嵌入在底层有限参数上的投影——截断投影。

拆开之后，"截断"不再是 FinStoch 内的限制，而是从无限维连续语义空间映射到有限维模型范畴时的信息损失。这需要定义上层范畴和底层范畴之间的映射关系——不是在原稿上修补措辞能完成的，需要新增一段数学设定。

## 附：已撤回的"硬伤"——收敛保证

原先将"收敛保证"列为第四项硬伤，认为该词在优化理论中有精准的狭窄含义，草稿的用法与之不匹配。经讨论后撤回该判定。

**撤回原因**：在物理和工程的日常交流中，"收敛"指"系统经过迭代优化达到稳定状态"——损失不降、参数更新趋零、行为可预测。这个用法在物理直觉上是融贯的，不依赖优化理论的定理承接。草稿在这里区分的东西是真实的（参数经过了迭代稳定化过程，上下文没有），"收敛"在日常用法层面承载这个区分是足够的。

**处理**：在 position paper 中保留"收敛"一词，但给一个操作化定义，并把"保证"换成"约束"。见 [收敛表述_定义段.md](收敛表述_定义段.md)。
